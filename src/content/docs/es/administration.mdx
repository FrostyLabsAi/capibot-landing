## Configuración de Proveedores LLM

Configura qué modelos de AI usar.

### Prioridad de Proveedores

```
LLM Provider Priority

When CapiBot needs AI, it tries providers in this order:

1. 🥇 Anthropic Claude
2. 🥈 Google Gemini
3. 🥉 Kimi (Moonshot)
4.    GLM (Zhipu AI)
5.    MiniMax
6.    Ollama (local)

[Add Provider] [Reorder]
```

**Cómo Funciona el Failover:**

1. El proveedor primario (Claude) intenta la solicitud
2. Si falla, automáticamente intenta el siguiente proveedor configurado
3. Continúa hasta éxito o que todos fallen

```

Notifications     Keep for [7 ▼] days

Comms/Messages    Keep for [15 ▼] days

```

**¿Por qué Auto-Cleanup?**

- Reduce tamaño de base de datos
- Mejora rendimiento
- Mantiene privacidad
- Controla costos

**Datos Importantes Nunca se Eliminan:**

- Tareas activas
- Memorias de agentes
- Base de conocimiento
- Información de empresa
- Configuración

## Mejores Prácticas

### Configuración

1. **Documenta Cambios** — Anota por qué cambiaste configuraciones
2. **Prueba Primero** — Intenta cambios en ambiente de desarrollo
3. **Respalda Primero** — Antes de cambios mayores
4. **Comienza Conservador** — Ajusta gradualmente
5. **Revisa Regularmente** — Auditoría mensual de configuración

### Seguridad

1. **Keys de API Fuertes** — Usa keys auto-generadas
2. **Rota Regularmente** — Cada 90 días
3. **Mínimo Privilegio** — Acceso mínimo requerido
4. **Monitorea Logs** — Observa actividad inusual
5. **Mantén Actualizado** — Última versión para parches de seguridad

### Rendimiento

1. **Tamaño Correcto del Pool** — Coincide conexiones de BD con uso
2. **Cache Inteligente** — No hagas cache de contenido dinámico
3. **Limpia Regularmente** — Auto-cleanup de datos viejos
4. **Monitorea Recursos** — Observa memoria y disco
5. **Optimiza Consultas** — Revisa logs de consultas lentas

## Solución de Problemas

**Configuración No se Guarda:**

- Revisa permisos
- Verifica que los valores sean válidos
- Revisa mensajes de error
- Intenta refrescar la página

**Prueba de Conexión Falla:**

- Verifica API keys
- Revisa conectividad de red
- Revisa reglas de firewall
- Revisa estado del servicio

**Issues de Rendimiento:**

- Reduce tamaño del pool de conexiones
- Reduce duración del cache
- Habilita auto-cleanup
- Revisa tamaño de base de datos

**Alertas de Seguridad:**

- Revisa logs de auditoría
- Revisa patrones de acceso
- Verifica allowlists
- Rota keys comprometidas
