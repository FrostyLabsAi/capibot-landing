# Monitoramento e Observabilidade

Mantenha-se informado sobre o que sua força de trabalho de IA está fazendo com ferramentas abrangentes de monitoramento.

### Tipos de Eventos

| Ícone | Tipo    | Descrição                           |
| ---- | ------- | ------------------------------------- |
| 📝   | Tarefa    | Criada, atualizada, completada, atribuída |
| 🤖   | Agente   | Criado, ativado, status alterado    |
| 💬   | Mensagem | Enviada, recebida                        |
| 📁   | Arquivo    | Criado, modificado, excluído            |
| 🔔   | Alerta   | Disparado, resolvido                   |
| 🏢   | Empresa | Criada, fase alterada, arquivada      |
| 🕐   | Cron    | Job executado                          |
| ⚡   | Webhook | Recebido, entregue                   |

### Filtragem

Filtre por:

- **Agente** — Atividade de agente específico
- **Empresa** — Eventos específicos de empresa
- **Tipo** — Tarefa, agente, mensagem, etc.
- **Intervalo de Tempo** — Última hora, dia, semana
- **Busca** — Palavra-chave em eventos

**Níveis de Log:**

- **DEBUG** — Diagnósticos detalhados

**Fontes:**

- Operações de agentes
- Gerenciamento de tarefas
- Chamadas de API
- Queries de banco de dados
- Eventos WebSocket

### Streaming ao Vivo

Quando ao vivo:

## Painel de Analytics

Métricas de performance e tendências.

**Acesso:** Barra de navegação → **Analytics**

### Métricas do Dashboard

**Performance de Agentes:**

- Tarefas completadas por agente
- Tempo médio de conclusão
- Taxas de sucesso/erro
- Custo por agente

**Métricas de Tarefas:**

- Tarefas por status
- Velocidade de conclusão
- Tempo médio em cada coluna
- Identificação de gargalos

**Métricas de Sistema:**

- Volume de chamadas de API
- Performance do banco de dados
- Conexões WebSocket
- Uso de memória

### Intervalos de Tempo

Veja dados para:

- Últimas 24 horas
- Últimos 7 dias
- Últimos 30 dias
- Intervalo personalizado

### Exportação

Baixe relatórios como:

- CSV (planilhas)
- JSON (dados brutos)
- PDF (relatórios formatados)

## Trilha de Auditoria

Logging de segurança e conformidade.

**Acesso:** Barra de navegação → **Audit Trail**

### O Que é Logado

Cada ação com:

- **Quem** — Usuário ou agente
- **O Que** — Ação realizada
- **Quando** — Timestamp
- **Onde** — Endereço IP
- **Como** — User agent (navegador/app)

**Categorias de Ação:**

- Autenticação (login, logout)
- Operações de tarefa (criar, atualizar, excluir)
- Operações de agente (criar, arquivar)
- Mudanças de configuração
- Ações de integração
- Eventos de webhook

## Boas Práticas

### Monitoramento

1. **Verifique Diariamente** — Revise atividade e métricas
2. **Observe Padrões** — Identifique tendências cedo
3. **Configure Alertas** — Seja notificado de problemas
4. **Revise Semanalmente** — Tendências de analytics

### Debugging

1. **Comece com Logs** — Encontre mensagens de erro
2. **Verifique Traces** — Veja o que agentes fizeram
3. **Revise Atividade** — Linha do tempo de eventos
4. **Trilha de Auditoria** — Quem fez o quê quando

### Performance

1. **Monitore Latência** — Tempos de resposta de agentes
2. **Observe Custos** — Uso de tokens por agente
3. **Acompanhe Taxas de Sucesso** — Métricas de qualidade
4. **Otimize Gargalos** — Operações lentas

## Solução de Problemas

**Nenhuma Atividade Mostrando:**

- Verifique se sistema está rodando
- Verifique se filtros não são muito restritivos
- Atualize a página

**Logs Não Atualizando:**

- Habilite modo "Ao Vivo"
- Verifique filtros de nível de log
- Pode precisar rolar

**Notificações Faltando:**

- Verifique configurações de notificação
- Verifique conexões de canal (Telegram, etc.)
- Verifique configurações de "Não Perturbar"

**Custos Altos:**

- Verifique Analytics para detalhamento
- Revise traces para operações caras
- Considere seleção de modelo
