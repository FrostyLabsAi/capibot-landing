# Hierarquia de Agentes

O CapiBot organiza agentes em uma hierarquia de três níveis. Entender esses níveis ajuda você a escolher o agente certo para o trabalho e construir equipes eficazes.

## Os Três Níveis

### Líder (CEO/Executivo)

**Líderes estratégicos com ampla autoridade**

Os agentes líderes são seus executivos. Eles podem:

- Criar planos de negócio e estratégias
- Contratar outros agentes para construir equipes
- Delegar trabalho em toda a organização
- Acessar todas as ferramentas e canais de comunicação
- Tomar decisões de alto nível

**Exemplos:**

- **Atlas** — CEO que elabora planos de negócio e orquestra operações
- **Helios** — CTO que lidera iniciativas técnicas
- **Aura** — CMO que supervisiona estratégia de marketing

**Quando usar:** Executando Empresas de IA, planejamento estratégico, coordenação multifuncional

### Especialista

**Especialistas de domínio com habilidades profundas**

Os agentes especialistas são seus membros de equipe profissionais. Eles:

- Têm expertise profunda em áreas específicas
- Executam tarefas complexas de forma independente
- Podem criar subagentes para trabalho paralelo
- Reportam-se a agentes Líderes
- Usam ferramentas avançadas dentro de seu domínio

**Exemplos:**

- **Nova** — Engenheira Full-Stack (constrói recursos end-to-end)
- **Pixel** — Designer UI/UX (cria interfaces e protótipos)
- **Echo** — Estrategista de Conteúdo (escreve cópia e conteúdo de marketing)
- **Scout** — Analista de Pesquisa (pesquisa de mercado e análise competitiva)

**Quando usar:** Trabalho técnico, projetos criativos, tarefas de pesquisa, qualquer trabalho especializado

### Estagiário

**Agentes juniores para tarefas mais simples**

Os agentes estagiários lidam com trabalho rotineiro e delimitado:

- Executam tarefas bem definidas
- Trabalham sob supervisão de Especialista
- Acesso limitado a ferramentas por segurança
- Ótimos para trabalho repetitivo
- Custo menor que Especialistas

**Exemplos:**

- Agentes de entrada de dados
- Assistentes de formatação de conteúdo
- Assistentes de teste
- Coletores de pesquisa

**Quando usar:** Tarefas simples e repetitivas, exercícios de aprendizado, operações sensíveis a custos

## Escolhendo o Agente Certo

### Para Tarefas Individuais

| Tipo de Tarefa          | Nível Recomendado | Exemplo                    |
| ----------------------- | ----------------- | -------------------------- |
| Escrever um blog post   | Especialista      | Echo (Estrategista de Conteúdo) |
| Design de landing page  | Especialista      | Pixel (Designer UI/UX)     |
| Pesquisar concorrentes  | Especialista      | Scout (Analista de Pesquisa) |
| Construir uma feature   | Especialista      | Nova (Engenheira Full-Stack) |
| Formatar documentos     | Estagiário        | DocFormatter               |
| Executar uma campanha   | Líder             | Atlas (CEO) + equipe       |

### Para Empresas de IA

**Equipe Mínima Viável:**

```
CEO (Líder)
└── 2-3 Especialistas
    └── Opcional: 1-2 Estagiários
```

**Exemplo de Empresa de Marketing:**

```
Atlas (CEO) — Estratégia e coordenação
├── Echo (Estrategista de Conteúdo) — Mensagens da campanha
│   └── ContentIntern — Rascunhos de blog posts
├── Pixel (Designer) — Assets visuais
└── Scout (Pesquisa) — Análise de mercado
```

## Criando Agentes

### O que é Criar?

Criar gera subagentes temporários para tarefas específicas:

- Agente pai delega trabalho
- Subagente trabalha independentemente
- Resultados reportam automaticamente
- Subagente desaparece quando termina

### Quando Criar vs Contratar

| Criar                    | Contratar                   |
| ------------------------ | --------------------------- |
| Tarefas paralelas únicas | Membro de equipe contínuo   |
| Capacidade temporária    | Papel permanente            |
| Processamento paralelo   | Colaboração entre projetos  |
| Trabalho exploratório    | Função central do time      |

**Exemplo:** Um Analista de Pesquisa cria 5 subagentes para analisar 5 concorrentes diferentes simultaneamente, depois compila os resultados.

## Agentes Personalizados

Além do roster pré-construído, você pode criar agentes personalizados:

### Quando Criar Agentes Personalizados

1. **Expertise Única** — Precisa de conhecimento especializado não no roster
2. **Voz da Marca** — Agente que conhece o tom da sua empresa
3. **Processo Proprietário** — Segue sua metodologia específica
4. **Otimização de Custo** — Agente mais leve para tarefas simples

### Componentes de Agente Personalizado

Cada agente tem um arquivo **SOUL.md** contendo:

- **Identidade** — Nome, papel, personalidade
- **Expertise** — Habilidades e áreas de conhecimento
- **Instruções de Operação** — Como eles abordam o trabalho
- **Estilo de Comunicação** — Tom e preferências de formatação

**Exemplo:** Crie um agente "Guardião da Marca" que garante que todo o conteúdo corresponda ao seu guia de estilo.

## Capacidades de Agentes

### O Que Todos os Agentes Podem Fazer

- Ler e escrever arquivos
- Buscar na web
- Enviar mensagens
- Criar e atualizar tarefas
- Acessar memória e conhecimento
- Comunicar com outros agentes

### O Que Agentes Líderes Podem Fazer (Extra)

- Submeter planos de negócio
- Contratar agentes para empresas
- Aprovar orçamentos (no modo autônomo)
- Orquestrar fluxos de trabalho multi-agente

### O Que Especialistas Podem Fazer (Extra)

- Usar ferramentas específicas de domínio
- Criar subagentes
- Revisar e aprovar trabalho
- Treinar agentes Estagiários

### Memória e Contexto

Cada agente tem memória persistente:

- **Memória Pessoal** — Notas e aprendizados entre conversas
- **Contexto da Empresa** — Conhecimento de sua Empresa de IA
- **Memória Compartilhada** — Informação acessível a todos os agentes
- **Memória de Sessão** — Contexto da conversa atual

Agentes lembram interações passadas e constroem sobre trabalho anterior.

## Trabalhando com Agentes

### Através do Mission Control

**Painel de Squad de Agentes:**

- Visualizar todos os agentes ativos
- Verificar status (ocioso, ocupado, offline)
- Criar novos agentes
- Acessar detalhes do agente

**Visão Detalhada do Agente:**

- Overview — Status, empresa, atividade recente
- SOUL — Personalidade e instruções
- Memória — Notas pessoais e contexto
- Tarefas — Trabalho atribuído
- Atividade — Histórico de ações
- Config — Configurações e overrides

### Através de Apps de Mensagens

Converse diretamente com qualquer agente:

**Telegram:**

```
Você: @Nova Construa um formulário de login para mim
Nova: Vou criar um formulário de login para você.
      Deve incluir opções de login social?
```

**Slack:**

```
Você: @Echo Escreva um anúncio de produto
Echo: Trabalhando nisso! Qual é a feature principal
      que devemos destacar?
```

**WhatsApp:**

```
Você: @Scout Pesquise nossos concorrentes
Scout: Vou analisar os 5 principais concorrentes
       no seu espaço. Verifique o Mission Control
       para o relatório completo em ~10 minutos.
```

## Status de Agente

Agentes mostram seu estado atual:

- **Idle** — Disponível e esperando trabalho
- **Active** — Trabalhando atualmente em uma tarefa
- **Busy** — Ocupado com trabalho em andamento
- **Offline** — Não está rodando atualmente
- **Error** — Encontrou um problema
- **Archived** — Não está em uso (pode ser reativado)

## Boas Práticas

1. **Combine Agente à Tarefa** — Use Especialistas para trabalho complexo, Estagiários para tarefas simples
2. **Comece Simples** — Comece com agentes individuais antes de criar empresas
3. **Construa Equipes com Cuidado** — Garanta habilidades complementares em equipes de empresa
4. **Monitore no Início** — Observe novos agentes de perto até confiar no trabalho deles
5. **Use Criação com Sabedoria** — Crie para trabalho paralelo, contrate para papéis contínuos
6. **Arquive Quando Concluir** — Mantenha seu squad limpo e gerenciável

## Padrões Comuns

### Padrão 1: Fluxo de Trabalho de Agente Único

```
Você → Agente → Resultado
```

Tarefas rápidas, controle direto

### Padrão 2: Handoff Sequencial

```
Pesquisador → Redator → Editor
```

Pipeline de fluxo de trabalho, portões de qualidade

### Padrão 3: Equipe Paralela

```
       ┌── Agente A ──┐
Você ──┼── Agente B ──┼── Resultados
       └── Agente C ──┘
```

Múltiplas perspectivas, execução rápida

### Padrão 4: Empresa de IA

```
Você → CEO → Equipe → Entregáveis
       ↑______↓
       (Reporta de volta)
```

Operação autônoma com supervisão
