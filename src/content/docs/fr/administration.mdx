## Paramètres des Fournisseurs LLM

Configurez quels modèles IA utiliser.

### Priorité des Fournisseurs

```
Priorité des Fournisseurs LLM

Quand CapiBot a besoin d'IA, il essaie les fournisseurs dans cet ordre :

1. 🥇 Anthropic Claude
2. 🥈 Google Gemini
3. 🥉 Kimi (Moonshot)
4.    GLM (Zhipu AI)
5.    MiniMax
6.    Ollama (local)

[Ajouter Fournisseur] [Réorganiser]
```

**Comment Fonctionne le Failover :**

1. Le fournisseur principal (Claude) tente la requête
2. Si échoue, essaie automatiquement le fournisseur configuré suivant
3. Continue jusqu'à succès ou tous échouent

## Bonnes Pratiques

### Configuration

1. **Documentez les Changements** — Notez pourquoi vous avez changé les paramètres
2. **Testez D'abord** — Essayez les changements dans l'environnement dev
3. **Sauvegardez D'abord** — Avant les changements majeurs
4. **Commencez Conservateur** — Resserrez progressivement
5. **Révisez Régulièrement** — Audit mensuel des paramètres

### Sécurité

1. **Clés API Fortes** — Utilisez des clés auto-générées
2. **Rotation Régulière** — Tous les 90 jours
3. **Privilège Minimal** — Accès minimum requis
4. **Surveillez les Logs** — Regardez l'activité inhabituelle
5. **Gardez à Jour** — Dernière version pour les correctifs de sécurité

### Performance

1. **Pool de Bonne Taille** — Faites correspondre les connexions DB à l'utilisation
2. **Cachez Sagement** — Ne cachez pas le contenu dynamique
3. **Nettoyez Régulièrement** — Nettoyage auto des anciennes données
4. **Surveillez les Ressources** — Regardez la mémoire et le disque
5. **Optimisez les Requêtes** — Vérifiez les logs de requêtes lentes

## Dépannage

**Paramètres qui ne Sauvegardent Pas :**

- Vérifiez les permissions
- Vérifiez que les valeurs sont valides
- Révisez les messages d'erreur
- Essayez de rafraîchir la page

**Échec du Test de Connexion :**

- Vérifiez les clés API
- Vérifiez la connectivité réseau
- Révisez les règles firewall
- Vérifiez le statut du service

**Problèmes de Performance :**

- Réduisez la taille du pool de connexions
- Réduisez la durée du cache
- Activez le nettoyage auto
- Vérifiez la taille de la base de données

**Alertes de Sécurité :**

- Révisez les logs d'audit
- Vérifiez les patterns d'accès
- Vérifiez les listes d'autorisation
- Faites des rotations des clés compromises
