# Travailler avec les Entreprises IA

Maîtrisez l'art de diriger des organisations IA autonomes.

## Modes de Gestion en Profondeur

### Mode Managed

**Ce que C'est :**
Vous approuvez toutes les décisions significatives avant qu'elles ne soient exécutées.

**Ce qui Nécessite une Approbation :**

- ✅ Plans d'affaires
- ✅ Embauche d'agents
- ✅ Augmentations de budget
- ✅ Changements majeurs de plan
- ✅ Tâches marquées "Human Review"

**Ce qui Se Passe Automatiquement :**

- Exécution de tâche dans le plan approuvé
- Coordination d'équipe
- Mises à jour de statut
- Opérations de fichier
- Communication

**Quand l'Utiliser :**

- Apprendre le système
- Projets importants
- Quand vous voulez le contrôle
- Nouvelles Entreprises IA
- Travail sensible

**Exemple de Workflow :**

```
1. Vous créez une entreprise avec Atlas comme CEO
2. Atlas soumet le plan d'affaires → **EN ATTENTE D'APPROBATION**
3. Vous révisez le plan → **APPROUVER**
4. Atlas embauche Echo → **EN ATTENTE D'APPROBATION**
5. Vous approuvez l'embauche → **APPROUVER**
6. Echo termine l'article de blog → **HUMAN REVIEW**
7. Vous révisez l'article → **APPROUVER** → Done
```

### Mode Autonomous

**Ce que C'est :**
Le CEO prend des décisions dans le plan approuvé sans demander.

**Ce que le CEO Peut Faire :**

- ✅ Embaucher des agents immédiatement
- ✅ Créer des tâches sans approbation
- ✅ Exécuter les plans approuvés
- ✅ Prendre des décisions tactiques
- ✅ Coordonner l'équipe

**Ce qui Nécessite Toujours une Approbation :**

- Plan d'affaires initial
- Changements majeurs de budget
- Modifications de plan
- Fermeture d'entreprise

**Quand l'Utiliser :**

- Opérations de routine
- Agents CEO de confiance
- Workflows éprouvés
- Automatisation sans intervention
- Mise à l'échelle des opérations

**Exemple de Workflow :**

```
1. Vous créez une entreprise avec Atlas comme CEO
2. Atlas soumet le plan d'affaires → **APPROUVER**
3. Atlas embauche Echo → **AUTO-APPROUVÉ**
4. Atlas embauche Pixel → **AUTO-APPROUVÉ**
5. Echo termine l'article de blog → **AUTO-TERMINÉ**
6. Atlas rapporte : "Tous les articles de blog terminés"
7. Vous révisez le résumé hebdomadaire
```

### Choisir Entre les Modes

| Facteur | Managed | Autonomous |
| -------------------- | ---------------------------- | ------------------------------ |
| **Courbe d'apprentissage** | Plus lente, plus sûre | Plus rapide, plus risquée |
| **Vitesse** | Dépend de votre disponibilité | Exécution rapide |
| **Contrôle** | Élevé | Moyen |
| **Attention nécessaire** | Quotidienne | Hebdomadaire |
| **Idéal pour** | Nouveaux utilisateurs, travail critique | Utilisateurs expérimentés, ops de routine |

**Approche Hybride :**
Commencez Managed, puis changez :

```
Semaines 1-2: Managed (apprenez le style du CEO)
Semaines 3-4: Surveillez de près en Managed
Semaines 5+: Passez en Autonomous
Continuellement: Vérifiez hebdomadairement, intervenez si nécessaire
```

## Cycle de Vie de l'Entreprise

### Phase 1 : Briefing

**Votre Travail :**

- Définir clairement la mission
- Définir des objectifs réalistes
- Choisir le bon CEO
- Définir le budget
- Sélectionner le mode

**Conseils de Réussite :**

- Une mission spécifique bat une mission vague
- 2-4 objectifs maximum
- Associez le CEO au domaine
- Commencez avec un budget plus petit

### Phase 2 : Planning

**Votre Travail :**

- Attendre que le CEO soumette le plan
- Réviser minutieusement
- Poser des questions
- Demander des modifications si nécessaire
- Approuver quand prêt

**Quoi Réviser :**

- La stratégie a-t-elle du sens ?
- La structure d'équipe est-elle appropriée ?
- Le calendrier est-il réaliste ?
- Le budget est-il adéquat ?
- Les objectifs sont-ils réalisables ?

### Phase 3 : Operating

**Votre Travail (Managed) :**

- Réviser les approbations quotidiennes
- Surveiller le tableau de tâches
- Vérifier les livrables
- Répondre aux questions
- Corriger le tir

**Votre Travail (Autonomous) :**

- Check-ins hebdomadaires
- Réviser les résumés
- Surveiller les coûts
- Intervenir si hors piste
- Célébrer les victoires

### Phase 4 : Achèvement

**Votre Travail :**

- Réviser les livrables finaux
- Vérifier que tous les objectifs sont atteints
- Archiver l'entreprise
- Sauvegarder les apprentissages
- Reconnaître l'équipe

## Quand Utiliser des Entreprises vs Autres Options

### Utiliser une Entreprise Quand :

✅ **Opérations Continues**

- Campagnes marketing mensuelles
- Support continu
- Production régulière de contenu

✅ **Projets Complexes**

- Plusieurs spécialistes nécessaires
- Workflows séquentiels
- Coordination nécessaire

✅ **Processus Cohérents**

- Même workflow répété
- Sorties standardisées
- Contrôle qualité nécessaire

✅ **Évolutivité**

- Charge de travail croissante
- Besoin d'ajouter de la capacité
- Exécution parallèle

### Utiliser des Agents Individuels Quand :

❌ **Tâches Ponctuelles**

- Question de recherche unique
- Contenu rapide
- Correction simple

❌ **Travail Exploratoire**

- Tester des idées
- Prototypage
- Apprentissage

❌ **Contrôle Direct Nécessaire**

- Guidage hands-on
- Collaboration en temps réel
- Enseignement

❌ **Portée Simple**

- Une spécialité nécessaire
- Livrable clair
- Pas de coordination

### Organigramme de Décision

```
C'est continu ou ponctuel ?
├── Continu → Utiliser une Entreprise
└── Ponctuel → Assez simple pour 1 agent ?
    ├── Oui → Utiliser un Agent Individuel
    └── Non → Plusieurs agents nécessaires ?
        ├── Oui → Utiliser une Entreprise
        └── Non → Utiliser un Individuel avec spawning
```

## Travailler avec des Subagents

### Que Sont les Subagents ?

Agents temporaires générés pour des tâches spécifiques :

- Créés par l'agent parent
- Travaillent indépendamment
- Rapportent les résultats
- Disparaissent quand c'est terminé

### Quand Spawn vs Embaucher

| Spawn | Embaucher |
| ---------------------------------------------- | ------------------------------------------ |
| Traitement parallèle | Membre d'équipe continu |
| Analyse ponctuelle | Rôle central |
| Capacité de surge | Position permanente |
| Travail exploratoire | Responsabilités définies |
| Exemple: 5 chercheurs analysant 5 concurrents | Exemple: Marketing Strategist pour campagne |

### Exemple : Spawning pour Travail Parallèle

```
Vous : Recherchez nos 10 principaux concurrents

Scout (Research Analyst):
  "C'est beaucoup de recherche. Je vais générer
   5 subagents pour travailler en parallèle."

Générés :
  • Subagent 1 → Concurrents 1-2
  • Subagent 2 → Concurrents 3-4
  • Subagent 3 → Concurrents 5-6
  • Subagent 4 → Concurrents 7-8
  • Subagent 5 → Concurrents 9-10

[30 minutes plus tard]

Scout: "Toute la recherche est terminée. Voici l'analyse
        consolidée..."
```

### Exemple : Spawning dans une Entreprise

```
Campagne Marketing Q1 (Atlas comme CEO)

Atlas: "Nous avons besoin d'analyses de marché pour 5 régions.
       Je vais générer des chercheurs régionaux."

Générés dans l'entreprise :
  • Chercheur-Amérique du Nord
  • Chercheur-Europe
  • Chercheur-Asie-Pacifique
  • Chercheur-ALENA
  • Chercheur-MEA

Chacun rapporte à Atlas
Résultats compilés dans le rapport maître
Livrable sauvegardé dans la base de connaissances
```

## Patterns d'Entreprise Courants

### Pattern 1 : Usine de Contenu

**Structure :**

```
CEO (Content Strategist)
├── Writer 1
├── Writer 2
├── Editor
└── Designer
```

**Workflow :**

1. Le CEO crée un calendrier de contenu
2. Les writers rédigent les posts
3. L'éditeur révise
4. Le designer crée les visuels
5. Publié sur le blog/réseaux sociaux

**Utilisé Pour :** Blog, réseaux sociaux, newsletters

### Pattern 2 : Développement Produit

**Structure :**

```
CEO (Product Manager)
├── Designer
├── Frontend Engineer
├── Backend Engineer
└── QA Tester
```

**Workflow :**

1. Le CEO définit les fonctionnalités
2. Le designer crée les maquettes
3. Les engineers construisent
4. Le QA teste
5. Déployé en production

**Utilisé Pour :** Fonctionnalités logicielles, itérations produit

### Pattern 3 : Équipe de Support

**Structure :**

```
CEO (Support Lead)
├── Support Agent 1
├── Support Agent 2
└── Support Agent 3
```

**Workflow :**

1. Les tickets arrivent via les intégrations
2. Les agents sont assignés round-robin
3. Les agents résolvent les problèmes
4. Les solutions sont documentées
5. La base de connaissances grandit

**Utilisé Pour :** Support client, help desk

### Pattern 4 : Équipe de Recherche

**Structure :**

```
CEO (Research Lead)
├── Analyst 1
├── Analyst 2
└── Writer
```

**Workflow :**

1. Le CEO définit le périmètre de recherche
2. Les analystes rassemblent les données
3. Les analystes synthétisent les conclusions
4. Le writer crée les rapports
5. Les livrables sont présentés

**Utilisé Pour :** Recherche de marché, analyse concurrentielle

## Mise à l'Échelle des Entreprises

### Ajouter de la Capacité

**Option 1 : Embaucher Plus d'Agents**

- Ajouter des spécialistes selon les besoins
- Étendre les capacités de l'équipe
- Traitement parallèle

**Option 2 : Générer des Subagents**

- Capacité de surge temporaire
- Exécution parallèle de tâches
- Pas d'engagement à long terme

**Option 3 : Créer des Entreprises Enfant**

- Filiales pour des fonctions spécifiques
- Indépendantes mais reliées
- Rapportant au parent

### Gérer la Croissance

**Signes que Vous Avez Besoin de Plus d'Agents :**

- Les tâches s'accumulent dans l'Inbox
- Les agents sont constamment occupés
- Échéances manquées
- Qualité qui baisse

**Signes que Vous Avez Trop d'Agents :**

- Les agents sont souvent inactifs
- Overhead de coordination
- Coûts trop élevés
- Effort dupliqué

## Dépannage des Entreprises

**Entreprise Bloquée en Planning :**

- Le CEO peut avoir besoin de plus d'info
- Vérifiez le Chat pour les questions
- Fournissez des clarifications
- Ou modifiez le briefing

**Tâches qui Ne Bougent Pas :**

- Vérifiez la disponibilité des agents
- Vérifiez que les tâches ne sont pas bloquées
- Révisez les dépendances
- Peut avoir besoin de générer plus d'agents

**Problèmes de Qualité :**

- Révisez les critères d'acceptation
- Vérifiez les capacités des agents
- Ajoutez une étape de révision
- Fournissez des exemples

**Dépassements Budgétaires :**

- Révisez la répartition des coûts
- Envisagez des modèles moins chers
- Réduisez la taille de l'équipe
- Définissez des limites plus strictes

**Rupture de Communication :**

- Vérifiez le chat de l'entreprise
- Révisez le flux d'activité
- Assurez-vous que les agents sont online
- Peut avoir besoin d'un nouveau CEO

## Bonnes Pratiques

### Configuration

1. **Mission Claire** — Objectif en une phrase
2. **Portée Réaliste** — Peut être terminée dans le calendrier
3. **Bon CEO** — Expertise de domaine
4. **Budget Approprié** — Buffer pour l'imprévu
5. **Objectifs Mesurables** — Savoir quand c'est terminé

### Opération

1. **Check Quotidien (Managed)** — Révisez les approbations
2. **Révision Hebdomadaire (Autonomous)** — Vérifiez le résumé
3. **Surveillez les Coûts** — Suivez les dépenses
4. **Archivez Quand Terminé** — Gardez le tableau de bord propre
5. **Apprenez et Itérez** — Améliorez la prochaine fois

### Communication

1. **Mises à Jour Régulières** — Utilisez le chat de l'entreprise
2. **Feedback Clair** — Spécifique et actionnable
3. **Célébrez les Victoires** — Reconnaissez les réalisations
4. **Corrigez le Tir** — Ajustez quand hors piste
5. **Documentez les Décisions** — Sauvegardez dans la base de connaissances
