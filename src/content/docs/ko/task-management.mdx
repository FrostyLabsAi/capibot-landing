# 작업 관리

작업 패널은 AI 인력을 통해 흐르는 작업에 대한 완전한 통제권을 제공합니다.

## 작업 보드 접근

**탐색 레일:** **작업** 클릭

각 상태에 대한 열이 있는 칸반 보드를 보게 됩니다.

## 작업 카드

각 카드는 다음을 표시합니다:

- **ID** — 고유 식별자 (T-42)
- **제목** — 간략한 설명
- **우선순위** — 색상 코드 중요도
- **담당자** — 작업 중인 사람
- **마감일** — 마감 시기
- **태그** — 카테고리 (있는 경우)

## 열 설명

### Inbox

시작 대기 중인 새 작업.

**여기서 무슨 일이 일어나나요:**

- 생성되었지만 할당되지 않은 작업
- 할당된 작업 에이전트가 시작하지 않음
- 미래 작업의 백로그

**사용자의 작업:**

- 준비되면 진행 중으로 끌어서 놓기
- 필요하면 재할당
- 우선순위 설정

### 진행 중

지금 활발히 진행 중인 작업.

**여기서 무슨 일이 일어나나요:**

- 에이전트가 작업을 체크아웃
- 적극적으로 작업 수행
- 진행 상황 업데이트 기록

**사용자의 작업:**

- 진행 상황 모니터링
- 에이전트의 질문에 답변
- 문제가 발생하면 차단됨으로 이동

### 검토

완료됨, 품질 확인 필요.

**여기서 무슨 일이 일어나요:**

- 에이전트가 작업 완료로 표시
- Aegis/Sentinel이 품질 평가
- 검토자가 승인하거나 변경 요청

**사용자의 작업:**

- 승인된 작업 검토
- 검토자 코멘트 확인

### Human Review

_(관리형 AI 기업만)_

완료됨, 승인 대기 중.

**여기서 무슨 일이 일어나요:**

- 작업이 품질 검토 통과
- 인간 승인 필요
- 승인할 때까지 Done으로 이동하지 않음

**사용자의 작업:**

- 산출물 검토
- 승인 또는 변경 요청 클릭

### 차단됨

의존성이나 문제로 진행할 수 없음.

**여기서 무슨 일이 일어나요:**

- 다른 작업 대기 중
- 에이전트가 질문이 있음
- 정보 누락
- 외부 차단

**사용자의 작업:**

- 차단자 세부 정보를 위해 코멘트 확인
- 의존성 해결하여 차단 해제
- 질문에 답변
- 필요하면 재할당

### 완료됨

완료되고 승인된 작업.

**여기서 무슨 일이 일어나요:**

- 작업 완전히 완료
- 산출물 저장됨
- 시간 기록됨
- 참조를 위해 사용 가능

## 작업 생성

### 방법 1: 빠른 생성

1. **"+ 새 작업"** 버튼 클릭
2. 기본 정보 입력:
   ```
   제목: AI 트렌드에 대한 블로그 포스트 작성
   설명: 2024년 상위 AI 트렌드를 다루는
        800단어 기사
   담당자: [Echo ▼]
   우선순위: [높음 ▼]
   마감일: [날짜 선택기]
   ```
3. **생성** 클릭

### 방법 2: 열에서

1. 열의 **"+"** 클릭
2. 작업이 해당 열에서 시작
3. 세부 정보 입력
4. 저장

### 방법 3: 메시지를 통해

```
사용자: @Atlas 작업 생성: 경쟁사 조사
Atlas: 작업 T-45 "경쟁사 조사" 생성됨
      담당자: Scout
      상태: Inbox
```

## 작업 이동

### 끌어서 놓기

카드를 열 간 클릭하고 끌기:

- Inbox → 진행 중
- 진행 중 → 검토
- 검토 → 완료됨 (승인된 경우)
- 모든 → 차단됨

### 상태 드롭다운

작업 카드의 상태 배지 클릭:

```
[진행 중 ▼]
  ├─ Inbox
  ├─ 진행 중
  ├─ 검토
  ├─ Human Review
  ├─ 차단됨
  └─ 완료됨
```

### 키보드 단축키

- **E** — 작업 편집
- **M** — 작업 이동
- **C** — 코멘트 추가
- **D** — 작업 삭제

## 필터링 및 검색

### 필터 바

다음으로 작업 필터:

- **상태** — 하나 또는 여러 열
- **담당자** — 특정 에이전트 또는 할당되지 않음
- **우선순위** — 긴급, 높음, 중간, 낮음
- **기업** — 특정 AI 기업
- **태그** — 카테고리
- **마감일** — 연체, 오늘, 이번 주

### 검색

다음으로 작업 검색:

- 제목 키워드
- 설명 콘텐츠
- 작업 ID
- 담당자 이름
- 코멘트

**전역 검색:** 어디에서나 Cmd/Ctrl + K

## 작업과 함께 작업하기

### 작업 할당

**담당자 변경:**

1. 작업 세부 정보 열기
2. 담당자 드롭다운 클릭
3. 에이전트 선택
4. 에이전트에게 알림

**자동 할당:**

- AI 기업은 기술에 따라 자동 할당
- 지원을 위한 라운드 로빈
- 가용성을 위한 로드 밸런싱

### 우선순위 설정

시각적 우선순위 시스템:

| 색상      | 우선순위 | 언제 사용                     |
| --------- | -------- | ----------------------------- |
| 🔴 빨간색 | 긴급     | 중요, 차단, 오늘 마감일       |
| 🟠 주황색 | 높음     | 중요함, 곧 마감               |
| 🟡 노란색 | 중간     | 정상 작업                     |
| 🟢 녹색   | 낮음     | 하고 싶은 것, 유연한 타이밍   |

**모범 사례:**

- 대부분의 작업은 중간이어야 함
- 높음/긴급은 아껴서 사용
- 매일 우선순위 재조정

### 의존성 관리

다음 경우 작업 의존성 설정:

- 작업 B는 작업 A가 완료될 때까지 시작할 수 없음
- 순차 워크플로우 필요
- 작업 간 리소스 공유

**의존성 설정:**

1. 작업 세부 정보 열기
2. 의존성 섹션으로 이동
3. "의존성 추가" 클릭
4. 차단 작업 선택
5. 작업이 자동으로 차단됨으로 표시

**예시:**

```
T-42: 경쟁사 연구 (먼저 완료해야 함)
   ↓
T-43: 경쟁 분석 작성 (T-42가 완료될 때까지 차단)
   ↓
T-44: 프레젠테이션 생성 (T-43이 완료될 때까지 차단)
```

### 코멘트 추가

**코멘트를 다는 이유:**

- 질문하기
- 피드백 제공
- 컨텍스트 공유
- 결정 기록

**방법:**

1. 작업 세부 정보 열기
2. 코멘트로 스크롤
3. 입력하고 전송
4. @이름으로 에이전트 언급

**알림:**

- 담당자가 새 코멘트 알림 받음
- 언급된 에이전트가 알림 받음
- 메시징 앱을 통해 답장 가능

### 파일 첨부

**산출물 업로드:**

1. 작업 세부 정보 열기
2. "파일 첨부" 클릭
3. 파일 선택 또는 드래그
4. 작업과 함께 저장된 파일

**자동 첨부:**

- 에이전트가 작업을 첨부할 수 있음
- 산출물이 지식 베이스에 자동 저장
- 작업 내역에 연결된 파일

## 사용 사례별 작업 워크플로우

### 콘텐츠 제작 파이프라인

```
연구 → 작성 → 편집 → 검토 → 게시
```

1. **연구** 작업 (Scout)
   - 정보 수집
   - 검토로 이동

2. **작성** 작업 (Echo)
   - 의존: 연구
   - 초안 생성
   - 검토로 이동

3. **편집** 작업 (Aegis)
   - 품질 검토
   - 승인하거나 변경 요청
   - (관리형인 경우) Human Review로 이동

4. **사용자 승인**
   - 최종 콘텐츠 검토
   - 승인 → 완료됨

5. **게시** 작업 예약
   - 자동으로 게시

### 기능 개발

```
디자인 → 개발 → 테스트 → 배포
```

1. **디자인** (Pixel)
   - 목업 생성
   - 검토 → 완료됨

2. **개발** (Nova)
   - 코드 구현
   - 디자인에 의존
   - 검토 → 완료됨

3. **테스트** (테스트 에이전트)
   - QA 테스트
   - 버그 → 개발으로 돌아감
   - 통과 → 완료됨

4. **배포** (예약됨)
   - 프로덕션으로 이동

### 지원 티켓

```
새로운 → 진행 중 → 해결됨
```

1. 고객 메시지 → 작업을 Inbox에 자동 생성
2. 지원 에이전트가 청구 → 진행 중으로 이동
3. 에이전트가 해결 → 검토로 이동
4. 자동 승인 → 완료됨
5. 고객에게 자동 알림

## 모범 사례

### 작업 생성

- **명확한 제목** — 제목만으로 작업 이해 가능해야 함
- **구체적인 기준** — "완료" 정의
- **적절한 크기** — 2-8시간이 이상적
- **한 명의 담당자** — 명확한 소유권
- **현실적인 날짜** — 품질을 위한 버퍼

### 작업 관리

- **매일 보드 검토** — 무슨 일이 일어나는지 알기
- **빠르게 차단 해제** — 작업이 중단되지 않도록
- **자주 우선순위 재조정** — 집중이 변경됨
- **완료된 작업 보관** — 보드를 깔끔하게 유지
- **태그 사용** — 필터 및 조직

### 에이전트와 함께 작업하기

- **질문에 답변** — 차단됨 열 확인
- **신속하게 검토** — 검토에서 병목 현상을 일으키지 마세요
- **피드백 제공** — 에이전트가 개선하도록 도움
- **신뢰하지만 확인** — 작업을 무작위로 점검

## 문제 해결

**Inbox에서 작업이 멈춤:**

- 에이전트가 오프라인일 수 있음
- 사용 가능한 에이전트에게 재할당
- 너무 큰지 확인 — 세분화

**진행 중에서 너무 오래 걸림:**

- 에이전트가 차단되었을 수 있음 — 코멘트 확인
- 작업이 너무 큼 — 더 작은 작업으로 분할
- 도움이 필요함 — 보조 에이전트 추가

**검토에서 너무 많음:**

- 검토자가 압도됨 — 더 많이 추가
- 품질 문제 — 표준 명확화
- 병목 — 검토 프로세스 조정 필요

**작업이 계속 반복됨:**

- 요구사항이 불분명 — 설명 업데이트
- 잘못된 에이전트 — 재할당
- 컨텍스트 누락 — 더 많은 세부 정보 추가
