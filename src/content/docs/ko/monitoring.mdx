# 모니터링과 관측성

포괄적인 모니터링 도구로 AI 인력이 무엇을 하는지 파악하세요.

## 활동 피드

모든 시스템 이벤트의 실시간 스트림.

**접근:** 탐색 레일 → **활동**

### 라이브 피드 보기

```
┌─────────────────────────────────────────────────────────┐
│ 활동                                        [필터]      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│ 방금                                                    │
│ 📝 Echo가 작업 T-42 완료 (블로그 포스트 편집)          │
│                                                         │
│ 2분 전                                                  │
│ 🤖 Nova가 작업 T-38 시작 (API 엔드포인트 구축)         │
│                                                         │
│ 5분 전                                                  │
│ 💬 Atlas로부터 메시지 수신                             │
│   "블로그 포스트 초안이 검토 준비 완료"               │
│                                                         │
│ 10분 전                                                 │
│ 📁 Pixel이 파일 생성: landing-page-design-v2.png       │
│                                                         │
│ 15분 전                                                 │
│ 🏢 Atlas가 Pixel을 Q1 마케팅 기업에 고용               │
│                                                         │
│ [더 로드...]                                           │
└─────────────────────────────────────────────────────────┘
```

### 이벤트 유형

| 아이콘 | 유형     | 설명                                  |
| ------ | -------- | ------------------------------------- |
| 📝     | 작업     | 생성, 업데이트, 완료, 할당            |
| 🤖     | 에이전트 | 생성, 활성화, 상태 변경               |
| 💬     | 메시지   | 전송, 수신                            |
| 📁     | 파일     | 생성, 수정, 삭제                      |
| 🔔     | 알림     | 트리거, 해결                          |
| 🏢     | 기업     | 생성, 단계 변경, 보관                 |
| 🕐     | Cron     | 작업 실행                             |
| ⚡     | 웹훅     | 수신, 전달                            |

### 필터링

다음으로 필터:

- **에이전트** — 특정 에이전트의 활동
- **기업** — 기업별 이벤트
- **유형** — 작업, 에이전트, 메시지 등
- **시간 범위** — 지난 시간, 하루, 주
- **검색** — 이벤트의 키워드

## 로그 뷰어

디버깅 및 모니터링을 위한 시스템 로그.

**접근:** 탐색 레일 → **로그**

### 로그 레벨

```
┌─────────────────────────────────────────────────────────┐
│ 로그                                        [라이브 ▶]  │
├─────────────────────────────────────────────────────────┤
│ 필터: [전체 레벨 ▼] [소스: 전체 ▼] [검색...]           │
├─────────────────────────────────────────────────────────┤
│                                                         │
│ 🔴 오류  14:32:01  에이전트 Nova가 DB 연결 실패        │
│ 🟡 경고  14:30:45  작업 T-42가 마감일에 가까움         │
│ 🔵 정보  14:28:12  에이전트 Echo가 성공적으로 생성     │
│ ⚪ 디버그 14:25:33  메모리 검색이 45ms에 완료          │
│                                                         │
│ [자동 스크롤: 켜짐]  [로그 다운로드]                   │
└─────────────────────────────────────────────────────────┘
```

**로그 레벨:**

- **오류** — 주목이 필요한 문제
- **경고** — 잠재적인 문제
- **정보** — 정상 작업
- **디버그** — 상세한 진단

**소스:**

- 에이전트 작업
- 작업 관리
- API 호출
- 데이터베이스 쿼리
- WebSocket 이벤트

### 라이브 스트리밍

**토글:** "라이브" 버튼 클릭

라이브일 때:

- 새 로그가 자동으로 나타남
- 하단으로 자동 스크롤
- 검사를 위해 일시정지
- 따라잡기 위해 재개

### 로그 검색

다음으로 검색:

- 로그 메시지 콘텐츠
- 소스 구성 요소
- 에이전트 이름
- 작업 ID
- 시간 범위

## 감사 추적

보안 및 규정 준수 로깅.

**접근:** 탐색 레일 → **감사 추적**

### 로깅되는 것

다음을 포함한 모든 작업:

- **누가** — 사용자 또는 에이전트
- **무엇을** — 수행된 작업
- **언제** — 타임스탬프
- **어디에서** — IP 주소
- **어떻게** — 사용자 에이전트 (브라우저/앱)

**작업 카테고리:**

- 인증 (로그인, 로그아웃)
- 작업 작업 (생성, 업데이트, 삭제)
- 에이전트 작업 (생성, 보관)
- 설정 변경
- 통합 작업
- 웹훅 이벤트

## 모범 사례

### 모니터링

1. **매일 확인** — 활동 및 메트릭 검토
2. **패턴 주시** — 조기에 트렌드 발견
3. **알림 설정** — 문제 알림 받기
4. **주간 검토** — 분석 트렌드

### 디버깅

1. **로그로 시작** — 오류 메시지 찾기
2. **추적 확인** — 에이전트가 한 일 보기
3. **활동 검토** — 이벤트 타임라인
4. **감사 추적** — 누가 언제 무엇을

### 성능

1. **지연 모니터링** — 에이전트 응답 시간
2. **비용 주시** — 에이전트별 토큰 사용량
3. **성공률 추적** — 품질 메트릭
4. **병목 최적화** — 느린 작업

## 문제 해결

**활동이 표시되지 않음:**

- 시스템이 실행 중인지 확인
- 필터가 너무 제한적이지 않은지 확인
- 페이지 새로고침

**로그가 업데이트되지 않음:**

- "라이브" 모드 활성화
- 로그 레벨 필터 확인
- 스크롤이 필요할 수 있음

**알림이 누락됨:**

- 알림 설정 확인
- 채널 연결 확인 (Telegram 등)
- "방해 금지" 설정 확인

**비용이 높음:**

- 분석에서 분석 확인
- 추적에서 비용이 많이 드는 작업 검토
- 모델 선택 고려
