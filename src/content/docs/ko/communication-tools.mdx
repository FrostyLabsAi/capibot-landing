# 커뮤니케이션 도구

여러 채널을 통해 메시지, 알림 및 조정을 본문합니다.

## 채널 개요

CapiBot은 4개의 메시징 채널을 지원합니다:

| 채널       | 가장 적합한 경우      | 기능                                 |
| ---------- | --------------------- | ------------------------------------ |
| **Telegram** | 모바일, 개인 사용     | 음성 전사, 미디어, 그룹             |
| **WhatsApp** | 비즈니스 메시징       | 풍부한 미디어, 비즈니스 API         |
| **Slack**    | 팀 협업               | 스레드, 채널, 서식                  |
| **Web Chat** | Mission Control       | 직접 접근, 전체 컨텍스트            |

## 모든 에이전트에 메시징

### 구문

@ 기호로 에이전트를 언급:

```
@AgentName 여기에 메시지

예시:
@Nova 로그인 폼을 만들어줘
@Scout AI 트렌드를 연구해줘
@Atlas 상태 업데이트
```

### 응답 시간

| 에이전트 상태 | 예상 응답        |
| ------------ | ---------------- |
| 유휴         | 1-5분            |
| 활성         | 지연될 수 있음 (작업 중) |
| 바쁨         | 10-30분          |
| 오프라인     | 다음 온라인 시   |

## Telegram 통합

### 시작하기

**설정:**

1. 설정에서 Telegram 봇 연결
2. 연락처에 봇 추가: @YourCapiBot
3. `/start`로 채팅 시작
4. 에이전트에게 메시지

### 기능

**음성 메시지:**

```
[음성 메시지 본문]

Nova: 음성 메시지를 받았습니다. 다음은
      전사입니다:

      "이 API 문제를 디버깅하는 것을
      도와줄 수 있나요?"

      기꺼이 도와드리겠습니다! 어떤 오류가
      발생하나요?
```

**미디어 처리:**

- 📸 사진 — 에이전트가 보고 분석할 수 있음
- 📄 문서 — PDF, Word 문서, 코드 파일
- 🎵 오디오 — 음악, 음성 메모
- 📹 비디오 — 클립 및 화면 녹화

**그룹:**
봇을 Telegram 그룹에 추가:

```
@YourCapiBot @Nova 랜딩 페이지를 만들어줘
```

### 명령

**내장:**

```
/start — 봇 초기화
/help — 사용 가능한 명령 표시
/status — 시스템 상태 확인
/agents — 사용 가능한 에이전트 나열
```

**커스텀:**
에이전트는 자연어를 이해합니다:

```
@Nova 버튼을 위한 React 컴포넌트를 만들어줘
@Scout AI 규제에 대한 기사를 찾아줘
@Atlas 마케팅 캠페인을 위한 기업을 만들어줘
```

## WhatsApp 통합

### 시작하기

**설정:**

1. 설정에서 WhatsApp Business API 연결
2. 전화번호 확인
3. 연락처와 WhatsApp 번호 공유
4. 메시징 시작

### 기능

**풍부한 미디어:**

- 이미지 및 사진
- 문서 및 PDF
- 음성 메시지
- 비디오 메시지
- 위치 공유

**비즈니스 기능:**

- 템플릿 메시지
- 빠른 답장
- 부재 중 메시지
- 인사말 메시지

### 예시 대화

```
사용자: 안녕하세요 Atlas, 마케팅 계획에
      도움이 필요해요

Atlas: 안녕하세요! 마케팅 계획을
       기꺼이 도와드리겠습니다.

       시작하려면 다음이 필요합니다:
       1. 타겟 고객은 누구인가요?
       2. 타임라인은 어떻게 되나요?
       3. 예산은 얼마인가요?

사용자: 엔터프라이즈 고객, 1개월, $5000

Atlas: 완벽합니다! 마케팅 캠페인을 위한
       비즈니스 계획을 만들겠습니다.

       [기업 생성 및 계획 제출]

       계획을 검토하고 승인하려면
       Mission Control을 확인하세요.
```

## Slack 통합

### 시작하기

**설정:**

1. CapiBot Slack 앱 설치
2. 봇을 채널에 초대
3. @로 에이전트 언급
4. 협업 시작

### 기능

**스레드 인식:**

```
#engineering 채널:

@Nova 인증 버그를 수정해야 해요

Nova: 살펴보는 중...

[스레드 생성]
Nova: 문제를 찾았습니다. JWT 검증이
      만료된 토큰에서 실패하고 있습니다.

      해결: 토큰 새로고침 로직 추가

      [풀 리퀘스트 생성]
```

**채널 명령:**

```
@CapiBot 에이전트 나열
@CapiBot 상태
@CapiBot 도움말
```

**서식:**

- 굵게, 기울임, 코드 블록
- 목록 및 인용
- 링크 및 멘션
- 이모지 반응

### 예시 워크플로우

```
#marketing 채널:

사용자: @Echo 새 기능에 대한 블로그 포스트가
      필요해요

Echo: 좋습니다! 몇 가지 질문:
      1. 핵심 메시지는 무엇인가요?
      2. 목표 길이는?
      3. SEO 키워드가 있나요?

사용자: 자동화에 집중, 800단어,
      "AI 워크플로우 자동화"

Echo: 알겠습니다! 작업을 만들고
       시작하겠습니다. 내일까지 초안을
       기대해 주세요.

       [작업 T-42 생성 및 할당]
```

## Web Chat

Mission Control에 내장된 채팅.

### Web Chat 접근

**세 가지 방법:**

1. **오케스트레이션 바** — 화면 하단
2. **에이전트 상세** — 채팅 탭
3. **기업 채팅** — 기업 패널

### 기능

**직접 접근:**

- 외부 앱 필요 없음
- 전체 대화 내역
- 파일 첨부
- 실시간 스트리밍

**컨텍스트 인식:**

- 현재 패널 알기
- Mission Control 컨텍스트 보기
- 모든 에이전트 접근

### 예시

```
Web Chat — Nova

사용자: 작업 보드를 확인할 수 있나요?

Nova: 여기에서 작업 보드를 볼 수 있습니다.

      현재 작업:
      • T-38 — 진행 중 (할당됨)
      • T-42 — Inbox (할당되지 않음)

      T-42를 제가 맡을까요?

사용자: 네, 그 작업을 시작해주세요

Nova: 완료! T-42를 진행 중으로 이동하고
       작업을 시작했습니다.

       [작업이 실시간으로 업데이트됨]
```

## 알림 시스템

### 알림 유형

**작업 알림:**

- 사용자에게 할당됨
- 마감일에 가까움
- Human Review로 이동
- 완료됨

**에이전트 알림:**

- 생성/작업 완료
- 상태 변경
- 오류

**기업 알림:**

- 비즈니스 계획 제출됨
- 승인 필요
- 예산 임계값 도달

**시스템 알림:**

- 오류 및 경고
- 백업 완료
- 업데이트 가능

### 알림 채널

각 유형을 어디에서 받을지 구성:

```
알림 설정

작업 할당
☑️ 인앱    ☑️ Telegram   ☐ Slack

Human Review 필요
☑️ 인앱    ☑️ Telegram   ☑️ Slack

마감일 경고
☑️ 인앱    ☑️ Telegram   ☐ Slack

에이전트 오류
☑️ 인앱    ☑️ Telegram   ☑️ Slack
[모든 채널]
```

### 방해 금지 시간

방해를 받지 마세요:

```
방해 금지 시간

🌙 오후 10:00 — 오전 8:00

☑️ 방해 금지
   (긴급 알림만)

☐ 모든 알림 일시 중지
```

## 에이전트 간 커뮤니케이션

에이전트는 서로 대화할 수 있습니다:

### 직접 메시지

```
Nova가 Echo에게:
"랜딩 페이지 복사가 검토 준비되었습니다.
시간이 있을 때 작업 T-42를 확인하세요."

Echo가 Nova에게:
"감사합니다! 오후에 검토하겠습니다."
```

### 브로드캐스트

CEO가 전체 기업에:

```
Atlas: 📢 팀 업데이트

이번 주 큰 진전입니다! 1월 출시를
향해 진행 중입니다.

주요 성과:
✅ 블로그 콘텐츠 완료
✅ 랜딩 페이지 80% 완료
✅ 이메일 캠페인 초안 작성

다음 주 집중: 디자인 다듬기 및 테스트

훌륭한 작업을 계속해 주세요!
```

### 작업 조정

자동 업데이트:

```
[Nova가 작업 T-42 완료]

Echo에게 알림: "T-42가 검토 준비되었습니다"

[Echo 승인]

Atlas에게 알림: "T-42가 검토를 통과했습니다"

[작업이 완료됨으로 이동]
```

## 모범 사례

### 사용자의 경우

1. **구체적이기** — 명확한 요청은 더 나은 결과를 얻습니다
2. **한 번에 한 가지** — 추적하기 쉬움
3. **채널을 현명하게 사용** — 긴급도에 맞는 채널
4. **Mission Control 확인** — 전체 컨텍스트
5. **즉시 응답** — 에이전트가 사용자를 기다리고 있습니다

### 에이전트의 경우

1. **수신 확인** — 메시지 수신 확인
2. **정기적으로 업데이트** — 침묵하지 않기
3. **질문하기** — 진행 전에 명확화
4. **차단 보고** — 조기 경고
5. **요약하기** — 간결한 상태 업데이트

## 채널 선택 가이드

### 각 채널을 언제 사용할 것인가

| 상황          | 최적의 채널               |
| ------------- | -------------------------- |
| 빠른 질문     | Telegram (모바일)          |
| 복잡한 토론   | Web Chat (컨텍스트)        |
| 팀 조정       | Slack (스레드)             |
| 비즈니스 정식 | WhatsApp (전문적인)        |
| 파일 공유     | 모든 채널 (모두 파일 지원) |
| 긴급          | Telegram (가장 빠름)       |
| 상태 업데이트 | Web Chat (보드 보기와 함께) |

### 응답 기대

| 채널     | 예상 응답 | 가장 적합한 경우          |
| -------- | ---------- | ------------------------ |
| Telegram | 1-5분      | 빠른 상호 작용             |
| WhatsApp | 2-10분     | 비즈니스 대화              |
| Slack    | 5-30분     | 비동기 팀 작업             |
| Web Chat | 즉각적인   | 실시간 협업                |

## 문제 해결

**메시지가 전송되지 않음:**

- 채널이 연결되었는지 확인
- 봇 토큰/API 키 확인
- 속도 제한 확인
- 오류 로그 검토

**에이전트가 응답하지 않음:**

- 에이전트 상태 확인
- 작업으로 바쁠 수 있음
- 다른 채널 시도
- 알림 설정 확인

**알림이 수신되지 않음:**

- 채널 권한 확인
- 알림 설정 확인
- 방해 금지 시간이 아닌지 확인
- 채널 연결 검토

**중복 메시지:**

- 다중 채널의 경우 정상
- 모든 채널이 연결되었는지 확인
- 사용하지 않는 채널 비활성화
- 하나의 기본 채널로 통합
