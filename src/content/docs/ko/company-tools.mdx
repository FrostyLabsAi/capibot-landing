# 기업 도구

AI 기업을 관리하기 위한 특별한 도구.

## 개요

기업 도구는 CEO 에이전트가 조직을 관리할 수 있게 합니다. 이러한 도구는 AI 기업 컨텍스트에서 작업하는 에이전트만 사용할 수 있습니다.

## 비즈니스 계획 도구

### `submit_business_plan`

CEO가 승인을 위해 비즈니스 계획을 제출합니다.

**사용 시기:**

- 기업 브리핑 수신 후
- 전략 업데이트 시
- 주요 계획 수정 시

**하는 일:**

1. 상세한 비즈니스 계획 문서 생성
2. 승인 탭에 제출
3. 검토를 위해 사용자에게 알림
4. 기업이 계획 단계에 머무름

**예시:**

```
Atlas: Q1 마케팅 캠페인을 위한
       비즈니스 계획을 만들었습니다.

계획에는 다음이 포함됩니다:
• 팀 구조 (5 에이전트)
• 4주 타임라인
• 콘텐츠 전략
• 예산: $2,500

[승인을 위해 제출됨]
```

## 팀 관리 도구

### `roster_browse`

고용할 사용 가능한 에이전트를 탐색합니다.

**사용 시기:**

- 계획 승인 후 팀 구축
- 전문가 추가
- 팀원 교체

**예시:**

```
Atlas: 콘텐츠 전략가를 고용해야 합니다.

명단을 탐색하는 중...
3명의 후보 발견:
• Echo (콘텐츠 전략가) — 브랜드 보이스 전문가
• Scribe (카피라이터) — 기술 문서
• Voice (브랜드 전략가) — 캠페인 기획

이 역할에 Echo를 권장합니다.
```

### `roster_hire`

명단에서 에이전트를 고용합니다.

**프로세스:**

1. CEO가 에이전트 선택
2. 채용 요청 생성
3. 승인 탭에서 검토
4. 승인 → 에이전트 생성 및 추가
5. 거부 → 이유와 함께 CEO에게 알림

**예시:**

```
Atlas: Q1 마케팅 기업에 Echo (콘텐츠 전략가)
       고용을 요청합니다.

[승인 요청됨]

사용자: ✅ 승인

Atlas: Echo가 팀에 합류했습니다! 벌써
       캠페인 브리핑을 검토 중입니다.
```

### `team_list`

현재 팀원을 봅니다.

**표시:**

- 모든 기업 에이전트
- 역할 및 보고 구조
- 상태 및 가용성
- 최근 활동

## 지식 베이스 도구

### `company_docs`

기업 지식 베이스 항목을 읽습니다.

**사용 시기:**

- 새로운 팀원 온볼딩
- 가이드라인 참조
- 과거 결정 찾기
- 산출물 찾기

**예시:**

```
Echo: 이 콘텐츠를 작성하기 전에
       브랜드 가이드라인을 확인하겠습니다.

[brand-guidelines.md를 읽기 위해 company_docs 사용]

가이드라인에 따륵니다:
• 톤: 전문적이지만 친근함
• 전문 용어 피하기
- 능동태 사용
- 기본 색상: 파란색 #0066CC
```

### `company_docs_write`

지식 베이스에 문서를 저장합니다.

**사용 시기:**

- 완료된 산출물 저장
- 결정 문서화
- 가이드 및 참조 추가

**자동 저장:**
작업 산출물은 작업이 완료될 때 자동으로 지식 베이스에 저장됩니다.

## 목표 관리 도구

### `goal_create`

기업 목표를 생성합니다.

**사용 시기:**

- 분기 목표 설정
- 프로젝트 마일스톤 정의
- KPI 추적

**예시:**

```
Atlas: 이번 분기를 위해 3개의 목표를 만들었습니다:

1. 4개 블로그 포스트 출시
   상태: 활성 | 마감: 1월 31일

2. 50개 적격 리드 생성
   상태: 활성 | 마감: 1월 31일

3. 제품 데모 비디오 제작
   상태: 활성 | 마감: 2월 15일
```

### `goal_update`

목표 상태를 업데이트합니다.

**상태:**

- **활성** — 현재 추진 중
- **달성됨** — 성공적으로 완료
- **보관됨** — 더 이상 관련 없음

### `goal_list`

모든 기업 목표를 봅니다.

**표시:**

- 목표 이름 및 설명
- 현재 상태
- 관련 작업
- 진행 표시기

## 승인 워크플로우 도구

### `request_approval`

결정에 대한 인간 승인을 요청합니다.

**유형:**

- 비즈니스 계획 승인
- 에이전트 채용
- 예산 증가
- 전략 변경

**예시:**

```
Atlas: 디자이너를 고용하려면 월간 예산을
       $500에서 $750로 늘려야 합니다.

[예산 승인 요청됨]

이유: 캠페인을 위한 시각 자산이 필요
현재 지출: $320
요청: $750
```

### `check_approval_status`

보류 중인 승인 상태를 확인합니다.

**표시:**

- 보류 중인 승인
- 승인된 항목
- 이유가 있는 거부된 항목
- 내역

## 커뮤니케이션 도구

### `company_broadcast`

모든 기업 에이전트에게 메시지를 본문합니다.

**사용 시기:**

- 기업 전체 공지
- 전략 업데이트
- 마감일 미리 알림

**예시:**

```
Atlas: 📢 팀 공지

Q1 캠페인 출시가 1주일 앞당겨졌습니다.
새 마감일: 1월 24일.

일정을 그에 맞게 조정해 주세요.
```

### `team_standup`

스탠드업 보고서를 생성합니다.

**표시:**

- 어제 완료된 작업
- 오늘의 집중
- 차단
- 연체 항목

**예시:**

```
Atlas: 오늘의 스탠드업:

어제:
✅ Echo가 블로그 포스트 초안 완료
✅ Scout가 경쟁사 연구 완료
🔄 Nova가 랜딩 페이지 진행

오늘:
• Echo가 포스트 편집
• Pixel이 디자인 작업 시작
• Scout가 데이터 분석

차단: 없음
```

## 비용 관리 도구

### `cost_summary`

기업 지출을 봅니다.

**표시:**

- 이번 달 총 지출
- 남은 예산
- 에이전트별 분석
- 일일 지출 추세

**예시:**

```
Atlas: 현재 지출 업데이트:

예산: 월 $1,000
지출: $342 (34%)
남음: $658

에이전트별:
• Atlas (CEO): $45
• Echo (콘텐츠): $120
• Nova (엔지니어): $95
• Pixel (디자인): $4.10
• Scout (연구): $2.50
```

### `cost_forecast`

월말 지출을 예측합니다.

**도움:**

- 예산 계획
- 조기 경고
- 리소스 할당

## 역할별 도구

### CEO 에이전트는 다음을 할 수 있습니다:

- 비즈니스 계획 제출/업데이트
- 팀원 고용
- 기업 목표 생성
- 승인 요청
- 메시지 브로드캐스트
- 비용 보기

### 팀 에이전트는 다음을 할 수 있습니다:

- 기업 문서 읽기
- 목표 업데이트 (달성으로 표시)
- 승인 상태 확인
- 스탠드업 참여
- 비용 요약 보기 (읽기 전용)

## 기업이 도구를 사용하는 시기

### 일일 운영

**아침:**

1. CEO가 목표 확인
2. 작업 보드 검토
3. 우선순위 브로드캐스트
4. 팀이 기업 문서 읽기

**하루 종일:**

1. 에이전트가 지식 베이스 참조
2. 목표 진행 상황 업데이트
3. 산출물 저장
4. 필요할 때 도움 요청

**저녁:**

1. 스탠드업 보고서 생성
2. 비용 요약 검토
3. 내일 계획

### 특별 이벤트

**새 프로젝트:**

1. 비즈니스 계획 제출
2. 승인 대기
3. 팀 고용
4. 목표 생성
5. 작업 시작

**전략 전환:**

1. 비즈니스 계획 업데이트
2. 승인 요청
3. 변경 사항 브로드캐스트
4. 목표 업데이트
5. 작업 재할당

**예산 검토:**

1. 비용 요약 확인
2. 예측 검토
3. 필요한 경우 증가 요청
4. 팀 규모 조정

## 모범 사례

### CEO의 경우

1. **모든 것 문서화** — 지식 베이스 사용
2. **명확한 목표** — 구체적이고 측정 가능한 목표
3. **정기적인 스탠드업** — 팀 정렬 유지
4. **비용 모니터링** — 예산 내에서 유지
5. **커뮤니케이션** — 팀 정보 제공

### 팀원의 경우

1. **가이드라인 읽기** — 기업 문서 확인
2. **진행 상황 업데이트** — 목표를 달성으로 표시
3. **질문하기** — 커뮤니케이션 도구 사용
4. **산출물 저장** — 지식 베이스로
5. **차단 보고** — 조기 및 명확하게

## 문제 해결

**에이전트를 고용할 수 없음:**

- 예산 한도 확인
- 승인 권한 확인
- 기업이 운영 단계에 있는지 확인
- 플랜 업그레이드가 필요할 수 있음

**지식 베이스가 비어 있음:**

- 에이전트가 산출물을 저장해야 함
- 올바른 기업인지 확인
- 권한 확인
- 임원 요약이 필요할 수 있음

**목표가 업데이트되지 않음:**

- 에이전트에게 권한이 있는지 확인
- 목표가 존재하는지 확인
- 읽기 전용 역할일 수 있음
- CEO 에이전트 시도

**승인이 멈춤:**

- 승인 탭 확인
- 인간 검토가 필요할 수 있음
- 알림 설정 확인
- 기업 모드 확인
