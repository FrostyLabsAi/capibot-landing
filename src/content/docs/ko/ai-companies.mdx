# AI 기업

AI 기업은 CapiBot의 가장 강력한 기능입니다. 이를 통해 원하는 만큼의 인간 감독으로 완전히 AI 에이전트로 운영되는 자율 조직을 운영할 수 있습니다.

## AI 기업이란 무엇인가요?

AI 기업은 CEO 에이전트가 이끄는 공유 목표를 위해 함께 작업하는 에이전트 그룹입니다:

- **CEO 에이전트** — 계획 생성, 팀원 채용, 작업 할당
- **팀 에이전트** — 작업을 실행하는 전문가
- **공유 목표** — 모두가 함께 노력하는 기업 전체 목표
- **지식 베이스** — 기업 정보의 중앙 저장소
- **예산 추적** — 기업 전체 비용 모니터링

직원들이 AI 에이전트인 스타트업이라고 생각하세요.

## 기업 라이프사이클

기업은 다섯 단계를 거칩니다:

### 1. 브리핑

기업이 무엇을 해야 할지 설명합니다:

- 기업 이름과 미션
- 초기 목표와 목적
- 예산 제약
- 관리 모드 (관리형 또는 자율)

![기업 개요 플레이스홀더]
_스크린샷: Mission Control의 기업 생성 양식_

### 2. 계획

CEO 에이전트가 상세한 비즈니스 계획을 생성합니다:

- 목표 달성을 위한 전략적 접근
- 필요한 팀 구조와 역할
- 타임라인과 마일스톤
- 리소스 요구사항

**진행하기 전에 검토하고 승인합니다.**

### 3. 승인됨

기업이 운영할 준비가 되었습니다:

- 비즈니스 계획이 승인됨
- CEO가 팀원 채용을 시작할 수 있음
- 예산이 할당됨
- 목표가 활성화됨

### 4. 운영 중

기업이 자율적으로 운영됩니다:

- CEO가 명단에서 에이전트를 고용
- 작업이 생성되고 할당됨
- 작업이 칸반 보드를 통해 흐름
- 산출물이 생성됨

### 5. 보관됨

기업 운영이 마무리됩니다:

- 모든 작업이 완료되거나 이전됨
- 지식 베이스가 보존
- 최종 비용 요약
- 나중에 다시 활성화하는 옵션

## 관리 모드

### 관리형 모드 (신규 사용자에게 권장)

관리형 모드에서는 통제권을 유지합니다:

- **비즈니스 계획**은 승인이 필요
- **에이전트 채용**은 승인이 필요
- **주요 작업**은 Human Review 열로 이동
- **예산 변경**은 승인이 필요

**가장 적합한 경우:** 시스템 학습, 중요한 프로젝트, 감독을 원할 때

### 자율 모드

자율 모드에서는 CEO가 독립적으로 운영합니다:

- 승인된 계획 내에서 결정
- 필요에 따라 에이전트 고용
- 중단 없이 작업 실행
- 주기적으로 진행 상황 보고

**가장 적합한 경우:** 일상적인 운영, 경험 있는 사용자, 핸즈오프 자동화

## AI 기업 대 개별 에이전트를 언제 사용할 것인가

### AI 기업을 사용할 때:

| 시나리오                    | 기업을 사용하는 이유                           |
| --------------------------- | ------------------------------------------- |
| 월간 마케팅 캠페인          | 일관된 프로세스, 매월 동일한 팀              |
| 지속적인 고객 지원          | 전담 지원 에이전트, 공유 지식               |
| 제품 개발                   | 엔지니어, 디자이너, QA가 함께 작업            |
| 콘텐츠 제작                 | 작가, 편집자, 연구원 협업                    |
| 시장 조사                   | 정의된 방법론을 가진 연구 팀                  |

### 개별 에이전트를 사용할 때:

| 시나리오                   | 개별 에이전트를 사용하는 이유             |
| -------------------------- | --------------------------------------- |
| 일회성 연구 질문           | 간단함, 조정이 필요 없음                |
| 빠른 콘텐츠 조각           | 기업 설정보다 빠름                      |
| 아이디어 탐색              | 기업 확정 전 테스트                     |
| 개인 비서 작업             | 간단한 작업에 대한 직접 통제            |

## 첫 번째 AI 기업 생성하기

### 1단계: 기업 패널 열기

Mission Control에서 왼쪽 사이드바의 **기업**을 클릭하세요.

### 2단계: 새 기업 생성

**"새 기업"** 버튼을 클릭하고 다음을 입력하세요:

- **이름** — 예: "Q1 마케팅 팀"
- **미션** — 기업이 달성할 것
- **CEO** — 사용 가능한 CEO 에이전트 중에서 선택 (Atlas와 같은)
- **관리 모드** — 안전을 위해 "관리형"으로 시작

### 3단계: 목표 정의

구체적이고 측정 가능한 목표를 추가하세요:

- "이번 달에 4개의 블로그 포스트 게시"
- "50개의 적격 리드 생성"
- "제품 데모 비디오 제작"

### 4단계: 브리핑 제출

CEO가 브리핑을 받고 계획을 시작합니다.

### 5단계: 계획 검토

CEO가 비즈니스 계획을 제출하면:

1. 접근 방식 검토
2. 제안된 팀 구조 확인
3. 타임라인과 예산 확인
4. **승인** 클릭 또는 변경 요청

### 6단계: 운영 모니터링

승인되면:

- 칸반 보드에 작업이 나타나는지 확인
- 지식 탭에서 산출물 검토
- 채용 요청이 들어올 때 승인
- 채팅 탭에서 기업과 대화

## 기업 구성 요소 이해하기

### 조직도

보고 구조를 보여주는 시각적 트리:

```
CEO (Atlas)
├── 마케팅 전략가 (Echo)
│   └── 콘텐츠 작가 (Nova)
├── 디자이너 (Pixel)
└── 연구 분석가 (Scout)
```

### 지식 베이스

기업별 정보:

- **임원 요약** — 미션과 전략 (모든 에이전트 컨텍스트에 주입)
- **링크** — 중요한 URL과 참조
- **가이드** — 방법 문서와 절차
- **산출물** — 완료된 작업이 여기에 자동 저장

### 목표 추적

각 목표는 다음을 보여줍니다:

- 상태 (활성, 달성됨, 보관됨)
- 진행 지표
- 관련 작업
- 완료 날짜

### 비용 모니터링

기업 전체 비용 추적:

- 에이전트별 비용 분석
- 모델별 토큰 사용량
- 월간 트렌드
- 예산 대 실제

## AI 기업과 함께 작업하기

### Mission Control을 통해

- **개요 탭** — 기업 상태, 단계, 통계
- **계획 탭** — 비즈니스 계획 보기 및 편집
- **팀 탭** — 조직도와 채용 워크플로우
- **작업 탭** — 기업별 칸반 보드
- **지식 탭** — 문서와 산출물
- **채팅 탭** — 기업 에이전트와의 직접 대화
- **활동 탭** — 기업이 수행한 모든 것
- **승인 탭** — 결정이 필요한 대기 중 항목
- **비용 탭** — 지출 분석

### 메시징 앱을 통해

기업에 메시지 본문:

```
사용자: @Q1 마케팅 팀 블로그 포스트는 어떻게 진행되나요?
CEO: 콘텐츠 작가가 초안을 완료했습니다.
      검토 상태입니다. 보시겠어요?
```

## 일반적인 워크플로우

### 마케팅 캠페인 시작

1. Atlas를 CEO로 "3월 캠페인" 기업 생성
2. 목표 설정: "100개 리드 생성" 및 "5개 블로그 포스트"
3. CEO의 계획 승인
4. CEO가 마케팅 전략가와 콘텐츠 작가 고용
5. 작업이 보드에 자동으로 나타남
6. 지식 탭에서 초안 검토
7. 최종 콘텐츠 승인
8. 캠페인 종료 후 기업 보관

### 고객 지원 운영

1. 영구 상태로 "지원 팀" 기업 생성
2. CEO가 지원 에이전트 고용
3. 지원 채널에 연결
4. 에이전트가 티켓을 작업으로 처리
5. 해결된 문제에서 지식 베이스 구축
6. 월간 성과 보고서 검토

## 모드 전환

언제든지 관리 모드를 변경할 수 있습니다:

**관리형 → 자율:**

- CEO의 결정에 자신감이 있을 때
- 신뢰할 수 있는 일상적인 운영의 경우
- 팀이 자신을 증명한 후

**자율 → 관리형:**

- 중요한 결정이 검토가 필요할 때
- 방향을 바꾸려 할 때
- 중요한 마일스톤 전에 검토

## 성공을 위한 팁

1. **관리형 모드로 시작** — 자율로 전환하기 전에 CEO가 어떻게 결정하는지 배우세요
2. **명확한 목표 설정** — 구체적이고 측정 가능한 목표는 에이전트가 집중하는 데 도움이 됩니다
3. **계획을 주의 깊게 검토** — CEO의 계획이 모든 것의 기초를 설정합니다
4. **지식 베이스 구축** — 미래 참조를 위해 중요한 문서 저장
5. **초기에 모니터링** — 처음 몇 가지 작업을 면접히 확인한 다음 감독 감소
6. **완료 시 보관** — 대시보드를 깔끔하게 유지, 나중에 다시 활성화 가능
