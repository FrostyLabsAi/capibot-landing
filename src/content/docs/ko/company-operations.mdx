# 기업 운영

AI 기업 관리는 CapiBot의 진정한 힘을 발휘하는 곳입니다. 이 가이드는 Mission Control을 통해 일상적인 기업 운영을 다룹니다.

## 기업 패널

탐색 레일에서 접근: **기업**

### 기업 목록 보기

모든 AI 기업 보기:

```
┌─────────────────────────────────────────────────────────┐
│ 기업                                        [+ 새 기업] │
├─────────────────────────────────────────────────────────┤
│ 필터: [전체 단계 ▼] [모드: 전체 ▼] [검색...]            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│ ┌─────────────────────────────────────────────────────┐ │
│ │ Q1 마케팅 캠페인                                    │ │
│ │ ● 운영 중 | 관리형 모드 | 예산: 월 $45              │ │
│ │                                                     │ │
│ │ 팀: Atlas + 4 에이전트   작업: 12 활성              │ │
│ │ [개요] [계획] [팀] [작업] [지식]                   │ │
│ └─────────────────────────────────────────────────────┘ │
│                                                         │
│ ┌─────────────────────────────────────────────────────┐ │
│ │ 지원 팀 (보관됨)                                    │ │
│ │ ○ 보관됨 | 자율 모드                                │ │
│ │                                                     │ │
│ │ [다시 활성화]                                       │ │
│ └─────────────────────────────────────────────────────┘ │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**카드 정보:**

- **이름** — 기업 식별자
- **단계** — 브리핑, 계획, 승인됨, 운영 중, 보관됨
- **모드** — 관리형 또는 자율
- **예산** — 현재 지출
- **팀 규모** — 에이전트 수
- **작업 수** — 활성 작업 항목

_스크린샷: 다른 단계에 있는 여러 기업을 보여주는 기업 목록_

### 단계 배지

기업 상태의 시각적 표시기:

- 🟡 **브리핑** — 설정 중, 계획 대기 중
- 🔵 **계획** — CEO가 비즈니스 계획 생성 중
- 🟢 **승인됨** — 계획 승인, 운영 준비 완료
- 🟣 **운영 중** — 적극적으로 실행 중
- ⚪ **보관됨** — 완료되었거나 일시 중지됨

## 기업 생성

### 단계별

1. **"+ 새 기업" 클릭**

2. **기업 세부 정보 입력:**

   ```
   이름: Q1 마케팅 캠페인
   미션: 엔터프라이즈 고객을 대상으로 제품 인지도 캠페인 출시
   
   CEO: [Atlas ▼]
   모드: [관리형 ▼]
   예산: 월 $100
   ```

3. **목표 설정:**

   ```
   목표 1: 4개 블로그 포스트 게시
   목표 2: 50개 적격 리드 생성
   목표 3: 제품 데모 비디오 제작
   ```

4. **브리핑 제출**

5. **CEO가 계획 생성** (보통 몇 분 내)

6. **계획 검토** — 다음 섹션 보기

## 기업 상세 보기

9개 탭이 있는 전체 대시보드를 볼 기업을 클릭하세요:

### 1. 개요 탭

한 눈에 기업 상태:

```

│ Q1 마케팅 캠페인                     [편집] [보관]      │
│ ● 운영 중 | 관리형 모드 | 14일 됨                      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│ 단계 진행:                                              │
│ 브리핑 → 계획 → 승인됨 → [운영 중] → 보관             │
│                                                         │
│ 통계:                                                   │
│ • 5 팀원     • 12 활성 작업                           │
│ • 8 작업 완료  • 월 $42 지출                          │
│                                                         │
│ CEO 상태: Atlas가 활성 중                              │
│ 마지막 활동: 2분 전                                    │
│                                                         │
│ 마일스톤:                                               │
│ ✅ 1단계 완료                                          │

│ ⏳ 3단계 대기 중                                       │

```

**빠른 작업:**

- 계획 보기
- 팀과 채팅

- 예산 조정

### 2. 계획 탭

비즈니스 계획 보기 및 관리:

```
┌─────────────────────────────────────────────────────────┐
│ 비즈니스 계획                                          │
├─────────────────────────────────────────────────────────┤
│ 상태: 승인됨 ✓                                          │
│ 제출자: Atlas (3일 전)                                  │
├─────────────────────────────────────────────────────────┤
│                                                         │
│ 임원 요약:                                              │
│ 콘텐츠 마케팅과 소셜 미디어를 통해                    │
│ 엔터프라이즈 고객을 대상으로 Q1 마케팅 캠페인 실행.     │
│                                                         │
│ 전략:                                                   │
│ 1. 콘텐츠 제작 (블로그 포스트, 사례 연구)             │
│ 2. 소셜 미디어 캠페인 (LinkedIn, Twitter)             │
│ 3. 게이트드 콘텐츠를 통한 리드 생성                   │
│                                                         │
│ 팀 구조:                                                │
│ • 마케팅 전략가 (Echo) — 캠페인 기획                  │
│ • 콘텐츠 작가 (Nova) — 블로그 포스트                  │
│ • 디자이너 (Pixel) — 시각 자산                        │
│ • 연구 분석가 (Scout) — 시장 분석                     │
│                                                         │
│ 타임라인: 4주                                           │
│ 예산: 월 $100                                           │
└─────────────────────────────────────────────────────────┘
```

**작업:**

- **계획 편집** — 승인 전 수정
- **승인** — 수락하고 운영 중으로 이동
- **변경 요청** — 피드백과 함께 CEO에게 다시 본문
- **내역 보기** — 계획 반복 보기

### 3. 팀 탭

조직도와 채용:

```
조직도

          Atlas (CEO)
             │
    ┌────────┼────────┬────────┐
    │        │        │        │
  Echo     Nova     Pixel    Scout
콘텐츠   엔지니어  디자이너  연구

[➕ 에이전트 고용] [명단 보기]
```

**기능:**

- 보고 구조의 시각적 트리 보기
- 에이전트를 클릭하여 세부 정보 보기
- 명단에서 새로운 에이전트 고용
- 팀원 제거
- 역할 재할당

**채용 워크플로우:**

1. CEO가 채용 요청 제출
2. 승인 탭에서 검토
3. 승인 → 에이전트 생성 및 추가
4. 거부 → 이유와 함께 CEO에게 알림

## 관리 모드 전환

### 관리형 → 자율

**준비가 되면:**

1. 기업 개요 탭으로 이동
2. "모드: 관리형" 드롭다운 클릭
3. "자율" 선택
4. 확인: "확실합니까? CEO가 승인 없이 결정을 내립니다."

**그런 다음 CEO는 다음을 할 수 있습니다:**

- 묻지 않고 에이전트 고용
- 자동으로 작업 생성
- 독립적으로 계획 실행
- 주기적으로 진행 상황 보고

### 자율 → 관리형

**감독을 원할 때:**

1. 기업 개요 탭으로 이동
2. "모드: 자율" 드롭다운 클릭
3. "관리형" 선택
4. 향후 모든 결정은 승인이 필요

## 기업 보관

기업이 미션을 완료하면:

1. **"보관" 버튼 클릭**
2. **옵션 선택:**
   - **보관만** — 모든 것 보존, 운영 중지
   - **보관 및 이전** — 완료되지 않은 작업을 다른 곳으로 이동
   - **삭제** — 완전히 제거 (주의해서 사용!)

3. **기업이 보관됨 상태로 이동**
   - 에이전트 해제 (재사용 가능)
   - 작업 동결
   - 지식 베이스 보존
   - 나중에 다시 활성화 가능

## 모범 사례

### 기업 설정

1. **명확한 미션** — 구체적이고 달성 가능한 목표
2. **적합한 CEO** — 도메인 전문성에 따라 선택
3. **현실적인 예산** — 작게 시작, 확장
4. **정의된 타임라인** — 일반적으로 2-8주
5. **측정 가능한 목표** — 구체적인 목표

### 기업 운영

1. **계획을 주의 깊게 검토** — 모든 것의 기초를 설정
2. **초기에 모니터링** — 처음 몇 가지 작업을 면접히 지켜보기
3. **지식 베이스 구축** — 중요한 문서 저장
4. **주간 비용 확인** — 예상치 못한 비용 피하기
5. **정기적으로 커뮤니케이션** — 업데이트를 위해 채팅 탭 사용

### 작업 승인

1. **시기적절한 검토** — Human Review에 작업을 방치하지 마세요
2. **명확한 피드백** — 변경을 위한 구체적인 요청
3. **결정 문서화** — 승인에 메모 추가
4. **신뢰하지만 확인** — 자율 작업을 무작위로 점검

## 일반적인 워크플로우

### 캠페인 시작

1. "3월 캠페인" 기업 생성
2. CEO로 Atlas 할당
3. 목표 설정: "100개 리드 생성"
4. Atlas가 팀 구조와 함께 계획 제출
5. 계획 승인
6. Atlas가 Echo (전략), Nova (콘텐츠), Pixel (디자인) 고용
7. 보드에 작업 나타남
8. 지식 탭에서 산출물 검토
9. 최종 콘텐츠 승인
10. 완료 시 기업 보관

### 지속적인 지원 운영

1. "고객 지원" 기업 생성
2. 영구 상태 설정 (종료 날짜 없음)
3. Atlas가 지원 에이전트 고용
4. 지원 채널에 연결
5. 에이전트가 티켓을 작업으로 처리
6. 해결책에서 지식 베이스 구축
7. 월간 분석 검토
8. 무기한 계속 실행

## 문제 해결

**계획에서 기업이 멈춤:**

- CEO가 더 많은 정보가 필요할 수 있음
- 질문을 위해 채팅 탭 확인
- 명확화 제공
- 또는 브리핑을 편집하고 다시 제출

**작업이 이동하지 않음:**

- 에이전트가 온라인인지 확인
- 작업이 차단되지 않았는지 확인
- 오류를 위해 활동 탭 확인
- 더 많은 에이전트를 생성해야 할 수 있음

**비용이 너무 높음:**

- 분석을 위해 비용 탭 검토
- 더 저렴한 LLM 모델 고려
- 에이전트 수 감소
- 더 엄격한 예산 한계 설정

**승인이 백업됨:**

- 정기적으로 승인 탭 검토
- 검토 시간 할당
- 자율로 전환 고려
- 또는 더 적은 에이전트 고용
