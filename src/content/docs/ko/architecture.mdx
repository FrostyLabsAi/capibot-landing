# 아키텍처 개요

CapiBot은 완전한 AI 인력 플랫폼으로 설계되었습니다. 구성 요소들이 어떻게 연결되는지 이해하면 효과적으로 사용할 수 있습니다.

## 상호 작용하는 세 가지 방법

### 1. Mission Control (웹 대시보드)

모든 것을 관리하는 명령 센터:

- 모든 에이전트와 상태 보기
- 칸반 보드에서 작업 관리
- 실시간으로 활동 모니터링
- 설정과 통합 구성
- 메모리와 지식 베이스 접근

**가장 적합한 경우:** 일상 관리, 작업 검토, 시스템 구성

### 2. 메시징 앱 (Telegram, WhatsApp, Slack)

AI 에이전트와 직접 채팅:

- 메시지 본문과 응답 받기
- 파일과 문서 공유
- 음성 메시지 전사 받기
- 알림 받기

**가장 적합한 경우:** 빠른 상호 작용, 이동 중 접근, 팀 협업

### 3. AI 기업 (자율 모드)

에이전트가 독립적으로 실행되도록 허용:

- CEO 에이전트가 비즈니스 계획 생성
- 팀이 자체 조직하고 실행
- 원할 때 인간 감독
- 자동 작업 위임

**가장 적합한 경우:** 복잡한 프로젝트, 지속적인 운영, 핸즈오프 자동화

## 핵심 구성 요소

### 에이전트

CapiBot은 전문별로 구성된 150개 이상의 사전 구축된 에이전트가 포함되어 있습니다:

- **CEOs** — 기업을 운영하는 전략적 리더
- **엔지니어** — 기능 개발 및 코드 작성
- **디자이너** — UI/UX 및 시각 자료 제작
- **마케터** — 캠페인 및 콘텐츠 처리
- **연구원** — 데이터 및 경쟁사 분석
- **그리고 더 많은...**

각 에이전트는 다음을 가지고 있습니다:

- 고유한 개성 (SOUL.md)
- 특정 기능
- 대화 간 지속되는 메모리
- 상태 추적 (유휴, 바쁨, 오프라인)

### 작업

작업은 칸반 보드를 통해 흐르는 방식으로 조직됩니다:

- **Inbox** — 시작 대기 중인 새 작업
- **In Progress** — 현재 작업 중
- **Review** — 품질 확인 준비 완료
- **Human Review** — 승인 필요
- **Blocked** — 무언가를 기다리는 중
- **Done** — 완료됨

### 메모리와 지식

AI는 모든 것을 기억합니다:

- **에이전트 메모리** — 각 에이전트의 개인 메모와 컨텍스트
- **지식 베이스** — 모든 에이전트가 접근 가능한 공유 정보
- **기업 지식** — 각 AI 기업에 특정한 정보
- **의미 검색** — 키워드뿐 아니라 의미로 정보 찾기

### 채널

선호하는 플랫폼을 통해 연결:

- **Telegram** — 음성 전사 기능이 있는 전체 기능 봇
- **WhatsApp** — 미디어 지원 비즈니스 메시징
- **Slack** — 스레드 인식 팀 협업
- **Web Chat** — Mission Control에 내장

## 데이터 흐름 방식

### 예시: 경쟁사 연구를 요청

1. **사용자**가 Telegram을 통해 메시지를 병냅니다: _"상위 5개 경쟁사를 조사하세요"_
2. **채널 서비스**가 메시지를 받고 라우팅합니다
3. **DeepAgent 서비스**가 연구 분석가 에이전트를 선택합니다
4. **에이전트**가 웹 검색 및 데이터 분석 도구를 사용합니다
5. **작업**이 생성되고 칸반 보드에서 추적됩니다
6. **결과**가 메모리에 저장되고 사용자에게 전송됩니다
7. **활동**이 검토를 위해 기록됩니다

### 예시: AI 기업이 마케팅 캠페인을 실행

1. **CEO 에이전트**가 캠페인을 위한 비즈니스 계획을 생성합니다
2. **사용자**가 Mission Control에서 계획을 승인합니다
3. **CEO**가 마케팅 전략가와 콘텐츠 작가를 고용합니다
4. **에이전트**가 자동으로 작업을 생성합니다
5. **작업**이 칸반 워크플로우를 통해 이동합니다
6. **산출물**이 기업 지식 베이스에 저장됩니다
7. **사용자**가 주요 결정을 검토하고 승인합니다

## 무엇을 언제 사용할 것인가

### 개별 에이전트를 사용할 때:

- 특정한 일회성 작업이 있을 때
- 작업에 대한 직접적인 통제를 원할 때
- 빠른 답변이나 연구가 필요할 때
- 아이디어를 탐색할 때

**예시:** _"Atlas, AI 트렌드에 대한 블로그 포스트를 작성하세요"_

### 여러 에이전트를 사용할 때:

- 작업이 여러 전문 분야에 걸쳐 있을 때
- 에이전트가 협업하도록 원할 때
- 다양한 관점이 필요할 때
- 작업을 병렬로 수행할 수 있을 때

**예시:** 연구 분석가가 데이터 수집, 콘텐츠 작가가 복사본 생성, 디자이너가 시각 자료 제작

### AI 기업을 사용할 때:

- 지속적인 운영이 있을 때
- 에이전트가 자체 조직하도록 원할 때
- 일관된 프로세스가 필요할 때
- 세부 관리 없이 확장하려 할 때

**예시:** 매월 캠페인을 자동으로 실행하는 마케팅 기업

### 서브에이전트를 사용할 때:

- 작업에 병렬 실행이 필요할 때
- 하위 작업을 위임하려 할 때
- 상위 에이전트가 조정에 집중해야 할 때
- 작업이 비동기적으로 발생할 수 있을 때

**예시:** CEO가 5명의 연구원을 생성하여 5명의 다른 경쟁사를 동시에 분석한 다음 결과를 컴파일
