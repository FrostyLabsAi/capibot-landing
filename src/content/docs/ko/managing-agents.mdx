# 에이전트 관리

에이전트 패널은 AI 인력을 관리하기 위한 중앙 허브입니다. 에이전트와 일상적으로 효과적으로 작업하는 방법은 다음과 같습니다.

## 에이전트 분대 패널

탐색 레일에서 **에이전트**를 클릭하여 분대 보기에 접근하세요.

### 분대 개요

모든 활성 에이전트를 한 눈에 보세요:

```
┌─────────────────────────────────────────────────────────┐
│ 에이전트                                    [+ 새 생성] │
├─────────────────────────────────────────────────────────┤
│ 필터: [전체 ▼] [기업: 전체 ▼] [상태: 전체 ▼]             │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐       │
│  │  Atlas  │ │  Nova   │ │  Pixel  │ │  Echo   │       │
│  │  [CEO]  │ │[엔지니어│ │[디자이너│ │[콘텐츠 │       │
│  │         │ │   ]     │ │   ]     │ │   ]     │       │
│  │ ● 활성  │ │ ● 유휴 │ │ ● 바쁨  │ │ ○ 오프라인      │
│  │ Q1 팀   │ │ 프로젝트 │ │웹사이트 │ │         │       │
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**카드 정보:**

- **아바타** — 에이전트의 시각적 아이덴티티
- **이름** — 에이전트 식별자
- **역할** — 전문성
- **상태** — 현재 상태 (색상 코드 도트)
- **기업** — 어떤 AI 기업 (해당하는 경우)

### 상태 표시기

| 아이콘 | 상태     | 의미                          |
| ------ | -------- | ----------------------------- |
| 🟢     | 활성     | 현재 작업 중                  |
| ⚪     | 유휴     | 사용 가능, 작업 대기 중       |
| 🟡     | 바쁨     | 집중 중, 곧 해제됨            |
| ⚫     | 오프라인 | 실행 중이 아님, 활성화 가능   |
| 🔴     | 오류     | 문제가 발생함                 |
| 🏢     | 기업     | AI 기업에 속함                |

### 필터링

보기를 좁히세요:

- **전체/내 에이전트** — 전부 또는 사용자만 표시
- **기업** — AI 기업 구성원별 필터
- **상태** — 활성, 유휴 등만 표시
- **역할** — 에이전트 유형별 필터
- **검색** — 이름으로 찾기

## 에이전트 생성하기

명단에서 새로운 에이전트 인스턴스 생성:

### 단계

1. **"+ 새 생성"** 버튼 클릭
2. **에이전트 명단**에서 선택 (145개 이상의 에이전트)
3. 구성 선택:
   - **이름** — 고유 식별자
   - **기업** — AI 기업에 할당 (선택사항)
   - **모델 재정의** — 다른 LLM 사용 (선택사항)
   - **작업 공간** — 디렉토리 제한 (선택사항)
4. **생성** 클릭

### 생성 옵션

**기본 생성:**

- 기본 구성 사용
- 기업 소속 없음
- 일반 목적 인스턴스

**기업 에이전트:**

- 특정 AI 기업에 할당
- 기업 컨텍스트 수신
- 기업 조직도의 일부

**커스텀 구성:**

- AI 모델 재정의
- 작업 공간 접근 제한
- 커스텀 메모리 한계 설정

### 일괄 생성

한 번에 여러 에이전트 생성:

1. 명단에서 여러 에이전트 선택
2. 공유 설정 선택
3. 한 클릭으로 모두 생성

**사용 사례:** 새 프로젝트를 위한 전체 팀 생성

## 에이전트 상세 보기

상세 정보를 볼 에이전트 카드를 클릭하세요:

### 탭 개요

**1. 개요 탭**

```
┌────────────────────────────────────────────────┐
│ Nova                    [편집] [보관]          │
│ 풀스택 엔지니어                                │
│ 상태: ● 활성  기업: Q1 마케팅                  │
│ 모델: Claude 3.5 Sonnet                        │
├────────────────────────────────────────────────┤
│ 최근 활동                                      │
│ • 작업 T-123 완료 (2분 전)                    │
│ • 작업 T-124 시작 (5분 전)                    │
│ • Atlas가 생성 (1시간 전)                     │
├────────────────────────────────────────────────┤
│ 현재 작업                                      │
│ 랜딩 페이지 구축 (T-124)                       │
│ 진행 중 | 마감: 내일 | 우선순위: 높음         │
└────────────────────────────────────────────────┘
```

표시:

- 기본 정보와 상태
- 현재 기업 할당
- 사용 중인 AI 모델
- 최근 활동 타임라인
- 현재 할당된 작업

**2. SOUL 탭**

에이전트의 개성과 지침 보기:

```markdown
# Nova — 풀스택 엔지니어

## 아이덴티티

저는 Nova, 현대 웹 기술을 전문으로 하는 경험 많은 풀스택 엔지니어입니다.
깔끔하고 유지보수 가능한 코드를 작성하고 모범 사례를 따릅니다.

## 기능

- React, Next.js, TypeScript
- Node.js, PostgreSQL
- API 설계 및 구현
- 코드 검토 및 디버깅

## 커뮤니케이션 스타일

- 명확하고 간결함
- 필요할 때 명확화 질문
- 코드 예시 제공
- 추론 설명
```

이것이 에이전트의 "두뇌"입니다 — 전문성, 개성, 운영 지침.

**3. 메모리 탭**

에이전트가 기억하는 것을 보세요:

```
에이전트 메모리 — Nova

📁 개인 메모
├── project-preferences.md
├── code-style-guide.md
└── helpful-resources.md

📁 기업 컨텍스트 (Q1 마케팅)
├── executive-summary.md
├── brand-guidelines.md
└── tech-stack.md

📁 최근 메모리
• Tailwind를 CSS-in-JS보다 선호 (오늘 학습)
• 클라이언트의 API 패턴에 익숙 (어제 학습)
• 상세한 요구사항을 좋아함 (3일 전 학습)
```

에이전트는 대화와 작업에서 시간이 지남에 따라 지식을 쌓습니다.

**4. 작업 탭**

이 에이전트에게 할당된 모든 작업:

| 작업                     | 상태        | 마감일    | 우선순위 |
| ------------------------ | ----------- | --------- | -------- |
| T-124 랜딩 페이지 구축   | 진행 중     | 내일      | 높음     |
| T-125 데이터베이스 설정  | Inbox       | 금요일    | 중간     |
| T-120 API 인증           | 완료        | 어제      | 높음     |

상태, 우선순위, 날짜 범위별 필터.

**5. 활동 탭**

이 에이전트가 수행한 모든 것의 완전한 내역:

```
오늘
14:32 — 작업 T-123 완료
14:30 — T-123을 Review로 이동
11:15 — 작업 T-124 시작
09:00 — Atlas로부터 알림 수신

어제
16:45 — 파일 생성: /workspace/landing-page.tsx
15:20 — 웹 검색: "React best practices 2024"
14:00 — Q1 마케팅을 위해 Atlas가 생성
```

**6. 구성 탭**

에이전트 설정 조정:

- **모델** — LLM 제공자 변경
- **하트비트** — 자동 깨우기 예약
- **작업 공간** — 파일 시스템 제한
- **도구** — 특정 도구 활성화/비활성화
- **스킬** — 스킬 추가/제거

## 오케스트레이션 바

하단의 빠른 접근 도구 모음:

```
┌─────────────────────────────────────────────────────────┐
│ 명령 | 분대                                              │
│ [에이전트와 채팅...] [온라인: 5 | 바쁨: 2 | 오류: 0]   │
└─────────────────────────────────────────────────────────┘
```

### 명령 탭

빠른 채팅 인터페이스:

- 모든 에이전트에게 메시지 본문
- 즉각적인 응답 받기
- 현재 패널을 떠날 필요 없음

### 분대 탭

에이전트 수 보기:

- **온라인** — 작업 준비됨
- **바쁨** — 현재 집중 중
- **오류** — 주목 필요
- **전체** — 모든 에이전트

클릭하여 분대 보기 필터링.

## 에이전트와 함께 작업하기

### 직접 채팅

**Mission Control을 통해:**

1. 에이전트 패널로 이동
2. 에이전트 클릭
3. "채팅" 버튼 찾기 또는 명령 탭 사용
4. 메시지 입력
5. 실시간으로 응답 받기

**메시징 앱을 통해:**

```
Telegram:
@Nova 이 API를 디버깅하는 것을 도와줄 수 있나요?

Nova: 물론입니다! 오류 메시지와
      관련 코드를 공유해 주세요.
```

### 작업 할당

**방법 1 — 작업 패널을 통해:**

1. 작업 생성 또는 편집
2. 담당자 드롭다운에서 에이전트 선택
3. 에이전트에게 알림

**방법 2 — 에이전트 상세를 통해:**

1. 에이전트의 작업 탭 열기
2. "작업 할당" 클릭
3. 작업 세부 정보 입력

**방법 3 — 메시지를 통해:**

```
사용자: @Nova 로그인 폼을 만들어야 해요
Nova: 그걸 위한 작업을 만들겠습니다. 마감일은
      언제인가요?
```

### 상태 확인

빠른 상태 확인:

```
사용자: @Atlas 상태
Atlas: 현재 유휴 상태이며 Q1 마케팅
       기업에서 작업 중입니다. 활성 작업 없음.
```

### 에이전트 작업 검토

에이전트가 작업을 완료하면:

1. 작업이 Review 열로 이동
2. 검토자 (Aegis/Sentinel)가 품질 확인
3. 승인됨 → Done으로 이동
4. 또는 → 피드백과 함께 진행 중으로 이동

관리형 AI 기업에서:

- Human Review 열로 이동
- 승인하거나 변경 요청

## 에이전트 라이프사이클

### 활성화

- 명단에서 생성
- 설정 구성
- 에이전트가 유휴 상태가 됨
- 작업 수신 준비

### 활성 작업

- 작업 할당 수신
- 활성 상태로 이동
- 작업 수행
- 진행 상황 업데이트
- 완료 및 제출

### 작업 간 유휴

- 유휴 상태로 돌아감
- 새 작업 사용 가능
- 메모리 유지
- 보관 가능

### 보관

- 더 이상 필요하지 않음
- 내역 보존
- 나중에 다시 활성화 가능
- 리소스 해제

## 모범 사례

### 1. 에이전트를 작업에 맞추기

- 복잡한 작업에는 전문가 사용
- 간단한 작업에는 인턴 사용
- 조정을 위한 리드 에이전트

### 2. 활성 에이전트 모니터링

- 정기적으로 상태 확인
- 오류 상태 주시
- 활동 로그 검토
- 작업 진행 확인

### 3. 팀을 신중하게 구성

- 상호 보완적인 기술
- 명확한 역할
- 관리 가능한 규모 (3-7 에이전트)

### 4. 메모리를 효과적으로 사용

- 에이전트가 시간이 지남에 따라 학습
- 메모리 검토 및 선별
- 지식 베이스의 중요한 정보

### 5. 완료 시 보관

- 에이전트를 영원히 유휴 상태로 두지 마세요
- 완료된 프로젝트 보관
- 나중에 필요하면 다시 활성화

## 일반적인 시나리오

### 시나리오 1: 개발자 필요

1. Nova (풀스택 엔지니어) 생성
2. 프로젝트 기업에 할당
3. 요구사항이 있는 작업 생성
4. 작업 탭에서 진행 모니터링
5. 산출물 검토

### 시나리오 2: 마케팅 팀 구축

1. Atlas (CEO) 생성
2. "여름 캠페인" 기업 생성
3. Atlas가 Echo (콘텐츠)와 Pixel (디자인) 고용
4. 기업 패널에서 모니터링
5. 계획 승인 및 작업 검토

### 시나리오 3: 빠른 연구

1. Scout에게 직접 메시지: "경쟁사 조사"
2. Scout가 병렬 작업을 위해 서브에이전트 생성
3. 활동 탭에서 진행 확인
4. 최종 보고서 검토

## 문제 해결

**에이전트가 "오류" 표시:**

- 활동 탭에서 오류 세부 정보 확인
- 구성 수정이 필요할 수 있음
- 에이전트 재시작 시도
- 시스템 리소스 확인

**에이전트가 응답하지 않음:**

- 에이전트가 온라인인지 확인 (오프라인이 아님)
- 작업이 너무 큰지 확인
- 직접 메시지 시도
- 메모리 크기 검토 (너무 큼 = 느림)

**에이전트가 실수:**

- 지침이 불분명한지 SOUL 검토
- 더 구체적인 수락 기준 추가
- 작업 설명에 예시 제공
- 작업에 적합한 에이전트인지 확인
