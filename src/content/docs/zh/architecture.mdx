## 三种交互方式

### 1. Mission Control（Web 仪表板）

管理一切的指挥中心：

- 查看所有 Agent 及其状态
- 在看板上管理任务
- 实时监控活动
- 配置设置和集成
- 访问记忆和知识库

**适用于：** 日常管理、审查工作、配置系统

### 2. 消息应用（Telegram、WhatsApp、Slack）

直接与您的 AI Agent 聊天：

- 发送消息并获取回复
- 分享文件和文档
- 获取语音消息转录
- 接收通知

**适用于：** 快速交互、随时随地访问、团队协作

### 3. AI 公司（自主模式）

让 Agent 独立运行：

- CEO Agent 制定商业计划
- 团队自我组织和执行
- 在您需要时进行人工监督
- 自动化任务委派

**适用于：** 复杂项目、持续运营、无需干预的自动化

## 核心组件

### Agent

CapiBot 配备 150+ 个预置 Agent，按专业领域组织：

- **CEO** — 运营公司的战略领导者
- **工程师** — 构建功能和编写代码
- **设计师** — 创建 UI/UX 和视觉效果
- **营销人员** — 处理活动和内容
- **研究员** — 分析数据和竞争对手
- **以及更多...**

每个 Agent 都有：

- 独特的个性（SOUL.md）
- 特定的能力
- 跨对话持续的记忆
- 状态跟踪（空闲、忙碌、离线）

### 任务

工作以任务形式组织，在看板上流转：

- **收件箱（Inbox）** — 等待开始的新任务
- **进行中（In Progress）** — 当前正在处理
- **审核中（Review）** — 准备进行质量检查
- **人工审核（Human Review）** — 需要您的批准
- **阻塞（Blocked）** — 等待某些条件
- **已完成（Done）** — 已完成

### 记忆与知识

您的 AI 记住一切：

- **Agent 记忆** — 每个 Agent 的个人笔记和上下文
- **知识库** — 所有 Agent 可访问的共享信息
- **公司知识** — 每个 AI 公司特有
- **语义搜索** — 通过含义而不仅是关键词查找信息

### 渠道

通过您偏好的平台连接：

- **Telegram** — 功能齐全的机器人，支持语音转录
- **WhatsApp** — 支持媒体的企业消息
- **Slack** — 支持线程的团队协作
- **Web 聊天** — 内置在 Mission Control 中

## 数据如何流动

### 示例：您让 Agent 研究竞争对手

1. **您** 通过 Telegram 发送消息：*"研究我们的前 5 名竞争对手"*
2. **渠道服务** 接收消息并路由
3. **DeepAgent 服务** 选择研究分析师 Agent
4. **Agent** 使用工具搜索网络和分析数据
5. **任务** 被创建并在看板上跟踪
6. **结果** 存储在记忆中并发送回给您
7. **活动** 被记录供您审查

### 示例：AI 公司运行营销活动

1. **CEO Agent** 为活动创建商业计划
2. **您** 在 Mission Control 中批准计划
3. **CEO** 聘请营销策略师和内容撰写者
4. **Agent** 自动创建任务
5. **任务** 在看板工作流中流转
6. **交付物** 保存到公司知识库
7. **您** 审查并批准关键决策

## 何时使用什么

### 何时使用个人 Agent：

- 您有特定的、一次性的任务
- 您希望对工作进行直接控制
- 您需要快速答案或研究
- 您在探索想法

**示例：** *"Atlas，写一篇关于 AI 趋势的博客文章"*

### 何时使用多个 Agent：

- 任务跨越不同专业领域
- 您希望 Agent 协作
- 您需要多样化的视角
- 工作可以并行进行

**示例：** 研究分析师收集数据，内容撰写者创建文案，设计师制作视觉效果

### 何时使用 AI 公司：

- 您有持续运营
- 您希望 Agent 自我组织
- 您需要一致的流程
- 您希望在无需微观管理的情况下扩展

**示例：** 一个每月自动运行活动的营销公司

### 何时使用子 Agent：

- 任务需要并行执行
- 您想委派子任务
- 父 Agent 需要专注于协调
- 工作可以异步进行

**示例：** CEO 生成 5 个研究员同时分析不同的竞争对手
