# 集成

将 CapiBot 连接到您已经在使用的工具。本指南涵盖设置 Composio 以实现 1000+ 应用集成，以及 GitHub 仓库管理。

## Composio 集成

Composio 提供对 1000+ 应用的统一访问，包括 Notion、Linear、Jira、Slack、Gmail、Airtable 等。

### 获取您的 Composio API 密钥

**第 1 步：注册 Composio**

1. 访问 [composio.dev](https://composio.dev)
2. 点击 **"Get Started"** 或 **"Sign Up"**
3. 使用您的邮箱或 GitHub 创建账户
4. 验证您的邮箱地址

**第 2 步：获取您的 API 密钥**

1. 登录您的 Composio 仪表板
2. 导航到 **Settings** → **API Keys**（或 **Developer** → **API Keys**）
3. 点击 **"Create New API Key"**
4. 命名：`CapiBot Integration`
5. 复制 API 密钥（以 `composio_` 开头）

**第 3 步：将 API 密钥添加到 CapiBot**

1. 打开 **Mission Control**（`http://localhost:3002`）
2. 转到 **Settings** → **Integrations**
3. 找到 **Composio** 部分
4. 在字段中粘贴您的 API 密钥：
   ```
   Composio API Key: composio_xxxxxxxxxxxxxxxx
   ```
5. 点击 **"Save"** 或 **"Connect"**
6. 您应该看到：✅ Connected

### 通过 Composio 连接应用

添加 Composio API 密钥后，您可以连接单个应用。

**第 1 步：浏览可用应用**

1. 在 Mission Control 中，转到 **Integrations** → **Composio**
2. 浏览或搜索应用（Notion、Linear、Slack 等）
3. 应用按类别组织（项目管理、通信、CRM 等）

**第 2 步：连接应用**

1. 找到您想要的应用（例如，Notion）
2. 点击 **"Connect"**
3. 您将被重定向到应用的 OAuth 授权页面
4. 登录您的应用账户（如果尚未登录）
5. 查看请求的权限
6. 点击 **"Authorize"** 或 **"Allow"**
7. 您将被重定向回 Mission Control
8. 应用现在显示：✅ Connected

**第 3 步：配置应用访问**
某些应用允许您配置 CapiBot 可以访问的内容：

**对于 Notion：**

- 选择要共享的页面/数据库
- 选择访问级别（只读或读写）
- 点击 **"Allow access"**

**对于 Slack：**

- 选择要连接的工作区
- 选择机器人可以访问的频道
- 授予权限（发送消息、读取消息等）

**对于 Linear：**

- 选择要连接的团队
- 设置问题访问权限
- 配置 webhook 设置

### 使用 Composio 连接的应用

连接后，Agent 可以通过 Composio 技能使用这些应用。

**示例：创建 Notion 页面**

```
You: Create a Notion page called "Project Ideas" with our
     meeting notes

Agent: [Uses composio skill]
       1. Checks Notion is connected ✅
       2. Finds "create page" action
       3. Creates page with your notes

       ✅ Created "Project Ideas" page in Notion
       View it: https://notion.so/...
```

**示例：发送 Slack 消息**

```
You: Send a message to the #engineering channel that the
     deployment is complete

Agent: [Uses composio skill]
       ✅ Message sent to #engineering:
       "Deployment complete! 🚀"
```

### 管理 Composio 连接

**查看连接的应用：**

1. 转到 **Integrations** → **Composio**
2. 查看所有连接的应用及其状态
3. 绿色勾选 = 已连接并正常工作
4. 红色 X = 连接问题

**重新连接应用：**
如果应用显示错误：

1. 点击 **"Reconnect"** 按钮
2. 重新授权应用
3. 如有需要更新权限

**断开应用：**

1. 在列表中找到应用
2. 点击 **"Disconnect"**
3. 确认："This will remove access to [App]"
4. CapiBot 将无法再使用该应用

### 常见 Composio 问题

**"No apps connected"**

- 您尚未连接任何应用
- 转到 Composio 标签并点击所需应用上的 "Connect"

**"App connection expired"**

- OAuth Token 在一段时间后过期
- 点击 "Reconnect" 刷新连接

**"Permission denied"**

- 应用已连接但缺少所需权限
- 断开并重新连接更广泛的权限
- 或在 Composio 仪表板中检查应用设置

**"Rate limit exceeded"**

- 短时间内 API 调用过多
- 等待几分钟再试
- 考虑升级您的 Composio 计划

## GitHub 集成

连接 GitHub 以同步问题、拉取请求和仓库活动与 CapiBot 任务。

### 获取您的 GitHub 个人访问令牌

**第 1 步：转到 GitHub 设置**

1. 登录您的 GitHub 账户
2. 点击您的头像（右上角）
3. 选择 **Settings**
4. 向下滚动并点击 **Developer settings**（左侧边栏）

**第 2 步：创建个人访问令牌**

1. 点击 **Personal access tokens** → **Tokens (classic)**
2. 点击 **"Generate new token"** → **"Generate new token (classic)"**
3. 您可能需要重新输入密码
4. 给您的令牌命名：`CapiBot Integration`
5. 设置过期时间（推荐：90 天）
6. 选择范围（权限）：

**必需范围：**

```
☑️ repo            (Full control of private repositories)
   ☑️ repo:status   (Access commit status)
   ☑️ repo_deployment  (Access deployment status)
   ☑️ public_repo   (Access public repositories)
   ☑️ repo:invite   (Access repository invitations)
☑️ admin:repo_hook  (Full control of repository hooks)
```

**可选范围（用于更多功能）：**

```
☑️ read:org        (Read org and team membership)
☑️ read:user       (Read user profile data)
☑️ read:project    (Read project boards)
```

7. 向下滚动并点击 **"Generate token"**
8. **重要：** 立即复制令牌！
   - 以 `ghp_` 开头
   - GitHub 只显示一次
   - 安全存储

**第 3 步：将令牌添加到 CapiBot**

1. 打开 **Mission Control**（`http://localhost:3002`）
2. 转到 **Settings** → **Integrations** → **GitHub**
3. 粘贴您的令牌：
   ```
   GitHub Personal Access Token: ghp_xxxxxxxxxxxxxxxx
   ```
4. 点击 **"Connect"**
5. 您应该看到：✅ Connected

### 设置 GitHub 同步

连接后，配置要同步的内容。

**第 1 步：选择仓库**

1. 转到 **Integrations** → **GitHub Sync**
2. 您将看到您有权访问的仓库列表
3. 选中要同步的仓库旁边的复选框
4. 点击 **"Save Selection"**

**第 2 步：配置同步设置**

```
Sync Mode: [Import issues as tasks ▼]
  • Import issues as tasks
  • Two-way sync (tasks ↔ issues)
  • Read-only (monitor only)

Filters:
  ☑️ Open issues only
  ☐ Include pull requests
  ☑️ Specific labels: [bug, feature, urgent]
  ☐ All labels

Auto-Assign Rules:
  When: Issue labeled "frontend"
  Assign to: @Pixel (Designer)

  When: Issue labeled "backend"
  Assign to: @Nova (Engineer)

  When: Issue labeled "bug"
  Priority: High
```

**第 3 步：测试同步**

1. 在您连接的仓库之一中创建一个测试问题
2. 添加与您的筛选匹配的标签（例如，"bug"）
3. 等待 30-60 秒
4. 检查 Mission Control 中的 **Tasks** 面板
5. 您应该看到问题作为任务导入

### GitHub 同步如何工作

**实时同步：**

```
GitHub Issue Created
        ↓
GitHub sends webhook to CapiBot
        ↓
CapiBot receives webhook
        ↓
Task T-123 created from issue
        ↓
Assigned based on label rules
        ↓
Agent gets notified
        ↓
Agent works on task
        ↓
Task completed in CapiBot
        ↓
GitHub issue closed (if two-way sync)
```

**同步历史：**
查看所有同步的项目：

```
Recent Imports

✅ #234 "Fix login bug" → T-123 (Nova)
   Synced: 2 min ago | Label: bug

✅ #235 "Add dark mode" → T-124 (Pixel)
   Synced: 5 min ago | Label: feature

⚠️ #236 "Update docs" → Skipped
   Reason: No matching label filter
```

### 管理 GitHub 同步

**暂停/恢复：**

- 暂时停止同步而不断开连接
- 维护期间有用
- 转到 GitHub Sync → 点击 "Pause"

**选择性同步：**
仅同步特定问题：

```
Labels to sync: [bug, feature, urgent]
Ignore: [wontfix, duplicate, question]
```

**批量导入：**
导入所有现有问题：

```
Import all open issues?
Repository: my-org/my-project
Issues found: 47

☑️ Yes, import all 47 issues
☐ Assign to default agent: @Nova
☐ Set priority based on labels
```

### 常见 GitHub 问题

**"Authentication failed"**

- 令牌可能已过期
- 生成新令牌
- 令牌可能缺少所需范围
- 检查范围并重新生成

**"Repository not found"**

- 您可能无法访问该仓库
- 检查仓库权限
- 尝试您拥有或是协作者的仓库

**"Webhook delivery failed"**

- CapiBot 可能无法公开访问
- 检查 GitHub 设置中的 webhook URL
- 确保 CapiBot 正在运行且可访问

**"Issues not syncing"**

- 检查标签筛选器是否匹配您的问题
- 验证同步是否启用（未暂停）
- 检查同步历史中的错误

### 添加新 API 密钥

**第 1 步：点击 "Add New Key"**

**第 2 步：选择集成类型**

- Composio
- GitHub
- Serper（Web Search）
- Groq（Voice Transcription）
- 自定义

**第 3 步：输入密钥详情**

```
Integration: Composio
Name: My Composio Key
Key: composio_xxxxxxxxxxxxxxxx
```

**第 4 步：保存并测试**

- 点击 **"Save"**
- 点击 **"Test Connection"** 验证
- 应该显示：✅ Connection successful

### 轮换密钥

**何时轮换：**

- 每 90 天（最佳实践）
- 如果您怀疑泄露
- 当员工离职时
- 安全事件后

**如何轮换：**

1. 在外部服务中生成新密钥
2. 转到 Settings → API Keys
3. 找到旧密钥
4. 点击 **"Regenerate"**
5. 粘贴新密钥
6. 点击 **"Save"**
7. 旧密钥立即失效

### 安全最佳实践

**对于 Composio：**

- 为 CapiBot 使用专用的 Composio 账户
- 不要使用您的个人 Composio 账户
- 定期审查连接的应用
- 移除未使用的应用连接

**对于 GitHub：**

- 尽可能使用细粒度令牌
- 将令牌过期限制为 90 天
- 仅授予最低所需权限
- 不要将令牌提交到仓库
- 在生产中使用环境变量

**通用：**

- 切勿在消息中分享 API 密钥
- 不要在截图中包含密钥
- 定期轮换密钥
- 监控异常使用
- 如果泄露立即撤销密钥

### 故障排除集成问题

**Connection Shows "Disconnected"**

1. 检查 API 密钥是否有效
2. 验证密钥是否具有所需权限
3. 手动测试连接
4. 如有需要重新输入密钥

**"Rate Limit Exceeded"**

1. 等待速率限制重置
2. 降低同步频率
3. 升级服务计划
4. 如果持续存在，联系支持

**"Webhook Delivery Failed"**

1. 检查 CapiBot 是否可公开访问
2. 验证 webhook URL 是否正确
3. 检查防火墙设置
4. 审查 webhook 日志

**"Sync Delayed"**

1. 检查集成健康
2. 验证同步是否启用
3. 查找错误消息
4. 尝试手动同步触发

## 集成模式

### 模式 1：GitHub + CapiBot 开发工作流

```
GitHub Issue Created → CapiBot Task Created
        ↓                        ↓
   Labeled "bug"          Assigned to @Nova
        ↓                        ↓
   Nova fixes bug         Task completed
        ↓                        ↓
   PR created             Deliverable attached
        ↓                        ↓
   PR merged              Task closed
        ↓                        ↓
   GitHub issue closed ←── Task done
```

### 模式 2：Composio 知识管理

```
Agent researches topic
        ↓
Findings saved to Notion (via Composio)
        ↓
Organized in company knowledge base
        ↓
Other agents reference Notion docs
        ↓
Knowledge grows over time
```

### 模式 3：多工具自动化

```
Cron job triggers daily
        ↓
Agent checks GitHub issues (via GitHub)
        ↓
Creates summary in Notion (via Composio)
        ↓
Sends Slack notification (via Composio)
        ↓
Team sees daily update
```

## 最佳实践

### 入门

1. **Start with One Integration** — 不要一次连接所有内容
2. **Test Thoroughly** — 依赖前验证每个集成是否正常工作
3. **Document Your Setup** — 记录密钥在何处使用
4. **Monitor Usage** — 定期检查集成健康

### 安全

1. **Use Dedicated Accounts** — 不要将个人账户用于集成
2. **Minimum Permissions** — 仅授予所需访问权限
3. **Regular Rotation** — 每 90 天轮换密钥
4. **Monitor Logs** — 注意异常活动

### 维护

1. **Health Checks** — 每周审查集成状态
2. **Update Apps** — 保持连接应用最新
3. **Clean Up** — 移除未使用的集成
4. **Backup Keys** — 安全离线存储
