# AI 公司

AI 公司是 CapiBot 最强大的功能。它们让您运行完全由 AI Agent 组成的自主组织，人工监督程度由您决定。

## 什么是 AI 公司？

AI 公司是一组朝着共同目标协作的 Agent，由 CEO Agent 领导：

- **CEO Agent** — 制定计划、招聘团队成员、分配工作
- **团队成员 Agent** — 执行任务的专家
- **共享目标** — 全公司共同追求的目标
- **知识库** — 公司信息的中央存储库
- **预算跟踪** — 监控全公司成本

将其视为一家初创公司，但员工是 AI Agent。

## 公司生命周期

公司经历五个阶段：

### 1. 简报（Briefing）

您描述希望公司做什么：

- 公司名称和使命
- 初始目标和目的
- 预算限制
- 管理模式（托管或自主）

### 2. 规划（Planning）

CEO Agent 制定详细的商业计划：

- 实现目标的战略方法
- 团队结构和所需角色
- 时间表和里程碑
- 资源需求

**您在继续前审查并批准。**

### 3. 已批准（Approved）

公司准备运营：

- 商业计划已批准
- CEO 可以开始招聘团队成员
- 预算已分配
- 目标已激活

### 4. 运营中（Operating）

公司自主运行：

- CEO 从名单中招聘 Agent
- 任务被创建和分配
- 工作在看板中流转
- 交付物被产出

### 5. 已归档（Archived）

公司运营逐步结束：

- 所有任务已完成或转移
- 知识库被保留
- 最终成本汇总
- 可选择稍后重新激活

## 管理模式

### 托管模式（Managed Mode）（新用户推荐）

在托管模式下，您保持控制：

- **商业计划** 需要您的批准
- **Agent 招聘** 需要您的同意
- **关键任务** 进入人工审核列
- **预算变更** 必须被批准

**适用于：** 学习系统、重要项目、需要监督时

### 自主模式（Autonomous Mode）

在自主模式下，CEO 独立运行：

- 在批准的计划内做出决策
- 根据需要招聘 Agent
- 无需中断执行任务
- 定期报告进度

**适用于：** 日常运营、有经验的用户、无需干预的自动化

## 何时使用 AI 公司与个人 Agent

### 何时使用 AI 公司：

| 场景 | 为什么使用公司 |
|------|---------------|
| 每月营销活动 | 一致的流程，每月同一团队 |
| 持续客户支持 | 专职支持 Agent，共享知识 |
| 产品开发 | 工程师、设计师、QA 协作 |
| 内容生产 | 撰稿人、编辑、研究员协作 |
| 市场研究 | 有明确方法论的研究团队 |

### 何时使用个人 Agent：

| 场景 | 为什么使用个人 Agent |
|------|---------------------|
| 一次性研究问题 | 简单，不需要协调 |
| 快速内容创作 | 比建立公司更快 |
| 探索想法 | 在投入公司前测试 |
| 个人助理任务 | 对简单任务的直接控制 |

## 创建您的第一个 AI 公司

### 第 1 步：打开公司面板

在 Mission Control 中，点击左侧边栏中的 **Companies**。

### 第 2 步：创建新公司

点击 **"New Company"** 按钮并填写：

- **名称** — 例如，"Q1 营销团队"
- **使命** — 公司将完成什么
- **CEO** — 从可用的 CEO Agent 中选择（如 Atlas）
- **管理模式** — 为安全起见从 "Managed" 开始

### 第 3 步：定义目标

添加具体、可衡量的目标：

- "本月发布 4 篇博客文章"
- "生成 50 个合格潜在客户"
- "创建产品演示视频"

### 第 4 步：提交简报

CEO 接收您的简报并开始规划。

### 第 5 步：审查计划

当 CEO 提交商业计划时：

1. 审查方法
2. 检查提议的团队结构
3. 验证时间表和预算
4. 点击 **Approve** 或请求更改

### 第 6 步：监控运营

一旦批准：

- 观看任务出现在看板上
- 在 Knowledge 标签中审查交付物
- 在出现招聘请求时批准
- 在 Chat 标签中与公司聊天

## 了解公司组件

### 组织架构图

显示汇报结构的可视化树：

```
CEO (Atlas)
├── 营销策略师 (Echo)
│   └── 内容撰写者 (Nova)
├── 设计师 (Pixel)
└── 研究分析师 (Scout)
```

### 知识库

公司特有的信息：

- **执行摘要** — 使命和战略（注入到所有 Agent 上下文中）
- **链接** — 重要的 URL 和参考
- **指南** — 操作文档和程序
- **交付物** — 完成的工作自动保存到这里

### 目标跟踪

每个目标显示：

- 状态（活跃、已达成、已归档）
- 进度指示器
- 相关任务
- 完成日期

### 成本监控

跟踪公司的支出：

- 每个 Agent 的成本分解
- 按模型的 Token 使用量
- 月度趋势
- 预算与实际情况

## 与您的 AI 公司协作

### 通过 Mission Control

- **Overview 标签** — 公司状态、阶段和统计
- **Plan 标签** — 查看和编辑商业计划
- **Team 标签** — 组织架构图和招聘工作流
- **Tasks 标签** — 公司特定的看板
- **Knowledge 标签** — 文档和交付物
- **Chat 标签** — 与公司 Agent 直接对话
- **Activity 标签** — 公司所做的一切
- **Approvals 标签** — 等待您的待处理决策
- **Costs 标签** — 支出分析

### 通过消息应用

向您的公司发送消息：

```
您：@Q1 营销团队 博客文章进展如何？
CEO：内容撰写者已完成草稿。
      它处于审核状态。您想看一下吗？
```

## 常见工作流

### 启动营销活动

1. 创建 "March Campaign" 公司，Atlas 担任 CEO
2. 设定目标："Generate 100 leads" 和 "5 blog posts"
3. 批准 CEO 的计划
4. CEO 聘请营销策略师和内容撰写者
5. 任务自动出现在板上
6. 在 Knowledge 标签中审查草稿
7. 批准最终内容
8. 活动结束后归档公司

### 运行客户支持

1. 创建 "Support Team" 公司，具有永久状态
2. CEO 聘请支持 Agent
3. 连接到您的支持渠道
4. Agent 将工单作为任务处理
5. 从已解决的问题中建立知识库
6. 审查月度绩效报告

## 切换模式

您可以随时更改管理模式：

**托管 → 自主：**

- 当您对 CEO 的决策有信心时
- 对于您信任的日常运营
- 在团队证明自己之后

**自主 → 托管：**

- 当重要决策需要审查时
- 如果您想改变方向
- 在关键里程碑前审查

## 成功技巧

1. **从托管模式开始** — 在转为自主之前了解 CEO 如何决策
2. **设定明确目标** — 具体、可衡量的目标帮助 Agent 专注
3. **仔细审查计划** — CEO 的计划为一切奠定基础
4. **建立知识库** — 保存重要文档供将来参考
5. **早期监控** — 密切观察前几个任务，然后减少监督
6. **完成后归档** — 保持仪表板整洁，您可以稍后重新激活
