# Agent 层级

CapiBot 将 Agent 组织成三层级结构。了解这些层级有助于您为工作选择合适的 Agent 并构建有效的团队。

## 三个层级

### 主管（Lead）（CEO/高管）

**具有广泛权力的战略领导者**

主管 Agent 是您的高管。他们可以：

- 制定商业计划和战略
- 招聘其他 Agent 组建团队
- 在整个组织中委派工作
- 访问所有工具和通信渠道
- 做出高层决策

**示例：**

- **Atlas** — 起草商业计划并协调运营的 CEO
- **Helios** — 领导技术项目的 CTO
- **Aura** — 监督营销策略的 CMO

**何时使用：** 运行 AI 公司、战略规划、跨职能协调

### 专家（Specialist）

**具有深厚技能的领域专家**

专家 Agent 是您的专业团队成员。他们：

- 在特定领域拥有深厚的专业知识
- 独立执行复杂任务
- 可以生成子 Agent 进行并行工作
- 向主管 Agent 汇报
- 在其领域内使用高级工具

**示例：**

- **Nova** — 全栈工程师（端到端构建功能）
- **Pixel** — UI/UX 设计师（创建界面和原型）
- **Echo** — 内容策略师（撰写文案和营销内容）
- **Scout** — 研究分析师（市场研究和竞争分析）

**何时使用：** 技术工作、创意项目、研究任务、任何专业工作

### 实习生（Intern）

**处理简单任务的初级 Agent**

实习生 Agent 处理日常的、有范围限制的工作：

- 执行明确的任务
- 在专家监督下工作
- 为安全起见有限制的工具访问
- 非常适合重复性工作
- 成本低于专家

**示例：**

- 数据录入 Agent
- 内容格式化助手
- 测试助手
- 研究收集员

**何时使用：** 简单、重复性任务、学习练习、成本敏感型运营

## 选择合适的 Agent

### 对于个人任务

| 任务类型 | 推荐层级 | 示例 |
|---------|---------|------|
| 撰写博客文章 | 专家 | Echo（内容策略师） |
| 设计落地页 | 专家 | Pixel（UI/UX 设计师） |
| 研究竞争对手 | 专家 | Scout（研究分析师） |
| 构建功能 | 专家 | Nova（全栈工程师） |
| 格式化文档 | 实习生 | DocFormatter |
| 运行活动 | 主管 | Atlas（CEO）+ 团队 |

### 对于 AI 公司

**最小可行团队：**

```
CEO（主管）
└── 2-3 个专家
    └── 可选：1-2 个实习生
```

**示例营销公司：**

```
Atlas（CEO）— 战略和协调
├── Echo（内容策略师）— 活动消息
│   └── ContentIntern — 起草博客文章
├── Pixel（设计师）— 视觉资产
└── Scout（研究）— 市场分析
```

## 生成 Agent

### 什么是生成？

生成创建用于特定任务的临时子 Agent：

- 父 Agent 委派工作
- 子 Agent 独立工作
- 结果自动回报
- 子 Agent 完成后消失

### 何时生成 vs 招聘

| 生成 | 招聘 |
|------|------|
| 一次性并行任务 | 持续的团队成员 |
| 临时激增能力 | 永久职位 |
| 并行数据处理 | 跨项目协作 |
| 探索性工作 | 核心团队职能 |

**示例：** 研究分析师生成 5 个子 Agent 同时分析 5 个不同的竞争对手，然后汇编结果。

## 自定义 Agent

除了预置名单外，您还可以创建自定义 Agent：

### 何时创建自定义 Agent

1. **独特专业知识** — 需要名单中不存在的专业知识
2. **品牌声音** — 了解您公司调性的 Agent
3. **专有流程** — 遵循您特定方法论
4. **成本优化** — 用于简单任务的轻量级 Agent

### 自定义 Agent 组件

每个 Agent 都有一个包含以下内容的 **SOUL.md** 文件：

- **身份** — 名称、角色、个性
- **专业知识** — 技能和知识领域
- **操作说明** — 他们如何开展工作
- **沟通风格** — 语气和格式偏好

**示例：** 创建一个 "Brand Guardian" Agent，确保所有内容符合您的风格指南。

## Agent 能力

### 所有 Agent 能做什么

- 读写文件
- 搜索网络
- 发送消息
- 创建和更新任务
- 访问记忆和知识
- 与其他 Agent 通信

### 主管 Agent 能做什么（额外）

- 提交商业计划
- 向公司招聘 Agent
- 批准预算（在自主模式下）
- 编排多 Agent 工作流

### 专家能做什么（额外）

- 使用领域特定工具
- 生成子 Agent
- 审查和批准工作
- 培训实习生

### 记忆和上下文

每个 Agent 都有持续的记忆：

- **个人记忆** — 跨对话的笔记和学习
- **公司上下文** — 来自其 AI 公司的知识
- **共享记忆** — 所有 Agent 可访问的信息
- **会话记忆** — 当前对话上下文

Agent 记住过去的互动并基于先前工作构建。

## 与 Agent 协作

### 通过 Mission Control

**Agent 团队面板：**

- 查看所有活跃的 Agent
- 检查状态（空闲、忙碌、离线）
- 生成新 Agent
- 访问 Agent 详情

**Agent 详情视图：**

- Overview — 状态、公司、最近活动
- SOUL — 个性和说明
- Memory — 个人笔记和上下文
- Tasks — 分配的工作
- Activity — 行动历史
- Config — 设置和覆盖

### 通过消息应用

直接与任何 Agent 聊天：

**Telegram：**

```
您：@Nova 帮我构建一个登录表单
Nova：我将为您创建一个登录表单。
      是否应该包含社交登录选项？
```

**Slack：**

```
您：@Echo 写一篇产品公告
Echo：正在写！我们应该重点介绍
      什么关键功能？
```

**WhatsApp：**

```
您：@Scout 研究我们的竞争对手
Scout：我将分析您所在领域的前 5 名竞争对手。
       大约 10 分钟后在 Mission Control 查看完整报告。
```

## Agent 状态

Agent 显示其当前状态：

- **空闲（Idle）** — 可用并等待工作
- **活跃（Active）** — 当前正在处理任务
- **忙碌（Busy）** — 正忙于正在进行的工作
- **离线（Offline）** — 当前未运行
- **错误（Error）** — 遇到问题
- **已归档（Archived）** — 未使用（可重新激活）

## 最佳实践

1. **Agent 与任务匹配** — 复杂工作用专家，简单任务用实习生
2. **从简单开始** — 在创建公司之前先使用个人 Agent
3. **深思熟虑地组建团队** — 确保公司团队中技能互补
4. **早期监控** — 密切关注新 Agent，直到您信任他们的工作
5. **明智使用生成** — 并行工作用生成，持续角色用招聘
6. **完成后归档** — 保持您的团队整洁且易于管理

## 常见模式

### 模式 1：单 Agent 工作流

```
您 → Agent → 结果
```

快速任务，直接控制

### 模式 2：顺序交接

```
研究员 → 撰写者 → 编辑
```

工作流管道，质量关卡

### 模式 3：并行团队

```
      ┌── Agent A ──┐
您 ──┼── Agent B ──┼── 结果
      └── Agent C ──┘
```

多种视角，快速执行

### 模式 4：AI 公司

```
您 → CEO → 团队 → 交付物
       ↑______↓
       (回报)
```

带监督的自主运营
