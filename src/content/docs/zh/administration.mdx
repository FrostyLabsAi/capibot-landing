## LLM 提供商设置

### 提供商优先级

```

When CapiBot needs AI, it tries providers in this order:

1. 🥇 Anthropic Claude
2. 🥈 Google Gemini
3. 🥉 Kimi (Moonshot)
4.    GLM (Zhipu AI)
5.    MiniMax
6.    Ollama (local)

[Add Provider] [Reorder]
```

**故障转移如何工作：**

1. 主提供商（Claude）尝试请求
2. 如果失败，自动尝试下一个配置的提供商
3. 继续直到成功或全部失败

## 最佳实践

### 配置

1. **Document Changes** — 记录更改设置的原因
2. **Test First** — 在开发环境中尝试更改
3. **Backup First** — 重大更改前
4. **Start Conservative** — 逐步收紧
5. **Review Regularly** — 每月设置审计

### 安全

1. **Strong API Keys** — 使用自动生成的密钥
2. **Rotate Regularly** — 每 90 天
3. **Least Privilege** — 最低所需访问
4. **Monitor Logs** — 注意异常活动
5. **Keep Updated** — 最新版本的安全补丁

### 性能

1. **Right-size Pool** — 数据库连接与使用匹配
2. **Cache Wisely** — 不要缓存动态内容
3. **Clean Regularly** — 自动清理旧数据
4. **Monitor Resources** — 注意内存和磁盘
5. **Optimize Queries** — 检查慢查询日志

## 故障排除

**Settings Not Saving：**

- 检查权限
- 验证值是否有效
- 审查错误消息
- 尝试刷新页面

**Connection Test Fails：**

- 验证 API 密钥
- 检查网络连接
- 审查防火墙规则
- 检查服务状态

**Performance Issues：**

- 降低连接池大小
- 减少缓存持续时间
- 启用自动清理
- 检查数据库大小

**Security Alerts：**

- 审查审计日志
- 检查访问模式
- 验证白名单
- 轮换泄露的密钥
