### 事件类型

| 图标 | 类型 | 描述 |
|------|------|------|
| 📝 | Task | 创建、更新、完成、分配 |
| 🤖 | Agent | 生成、激活、状态更改 |
| 💬 | Message | 发送、接收 |
| 📁 | File | 创建、修改、删除 |
| 🔔 | Alert | 触发、解决 |
| 🏢 | Company | 创建、阶段更改、归档 |
| 🕐 | Cron | 任务执行 |
| ⚡ | Webhook | 接收、交付 |

### 筛选

按以下筛选：

- **Agent** — 特定 Agent 的活动
- **Company** — 公司特定事件
- **Type** — 任务、Agent、消息等
- **Time Range** — 最近一小时、一天、一周
- **Search** — 事件中的关键词

## Analytics 面板

性能指标和趋势。

**访问：** 导航栏 → **Analytics**

### 仪表板指标

**Agent 性能：**

- 每个 Agent 完成的任务
- 平均完成时间
- 成功/错误率
- 每个 Agent 的成本

**任务指标：**

- 按状态的任务
- 完成速度
- 每列平均时间
- 瓶颈识别

**系统指标：**

- API 调用量
- 数据库性能
- WebSocket 连接
- 内存使用

### 时间范围

查看以下数据：

- 最近 24 小时
- 最近 7 天
- 最近 30 天
- 自定义范围

### 导出

下载报告为：

- CSV（电子表格）
- JSON（原始数据）
- PDF（格式化报告）

## 审计追踪

安全和合规日志。

**访问：** 导航栏 → **Audit Trail**

### 记录什么

每个动作包括：

- **Who** — 用户或 Agent
- **What** — 执行的操作
- **When** — 时间戳
- **Where** — IP 地址
- **How** — 用户代理（浏览器/应用）

**动作类别：**

- 认证（登录、注销）
- 任务操作（创建、更新、删除）
- Agent 操作（生成、归档）
- 设置更改
- 集成操作
- Webhook 事件

## 最佳实践

### 监控

1. **Check Daily** — 审查活动和指标
2. **Watch for Patterns** — 及早发现趋势
3. **Set Alerts** — 获取问题通知
4. **Review Weekly** — 分析趋势

### 调试

1. **Start with Logs** — 查找错误消息
2. **Check Traces** — 查看 Agent 做了什么
3. **Review Activity** — 事件时间线
4. **Audit Trail** — 谁做了什么何时

### 性能

1. **Monitor Latency** — Agent 响应时间
2. **Watch Costs** — 每个 Agent 的 Token 使用
3. **Track Success Rates** — 质量指标
4. **Optimize Bottlenecks** — 慢操作

## 故障排除

**No Activity Showing：**

- 检查系统是否正在运行
- 验证筛选器是否限制过多
- 刷新页面

**Logs Not Updating：**

- 启用 "Live" 模式
- 检查日志级别筛选器
- 可能需要滚动

**Missing Notifications：**

- 检查通知设置
- 验证渠道连接（Telegram 等）
- 检查 "Do Not Disturb" 设置

**High Costs：**

- 检查 Analytics 了解分解
- 审查追踪了解昂贵操作
- 考虑模型选择
