## 创建公司

### 分步指南

1. **点击 "+ New Company"**

2. **填写公司详情：**

   ```
   Name: Q1 Marketing Campaign
   Mission: Launch product awareness campaign
            targeting enterprise customers

   CEO: [Atlas ▼]
   Mode: [Managed ▼]
   Budget: $100/month
   ```

3. **设定目标：**

   ```
   Goal 1: Publish 4 blog posts
   Goal 2: Generate 50 qualified leads
   Goal 3: Create product demo video
   ```

4. **提交简报**

5. **CEO 创建计划**（通常在几分钟内）

6. **审查计划** — 见下一节

## 切换管理模式

### 托管 → 自主

**准备就绪时：**

1. 转到公司 Overview 标签
2. 点击 "Mode: Managed" 下拉菜单
3. 选择 "Autonomous"
4. 确认："Are you sure? CEO will make decisions without approval."

**然后 CEO 可以：**

- 无需询问即可招聘 Agent
- 自动创建任务
- 独立执行计划
- 定期报告进度

### 自主 → 托管

**当您想要监督时：**

1. 转到公司 Overview 标签
2. 点击 "Mode: Autonomous" 下拉菜单
3. 选择 "Managed"
4. 所有未来决策都需要批准

## 归档公司

当公司完成其使命时：

1. **点击 "Archive" 按钮**
2. **选择选项：**
   - **Archive Only** — 保留一切，停止运营
   - **Archive & Transfer** — 将未完成的任务转移到其他地方
   - **Delete** — 完全删除（谨慎使用！）

3. **公司进入 Archived 状态**
   - Agent 被释放（可以重用）
   - 任务被冻结
   - 知识库被保留
   - 可以稍后重新激活

## 最佳实践

### 设置公司

1. **Clear Mission** — 具体、可实现的目标
2. **Right CEO** — 基于领域专业知识选择
3. **Realistic Budget** — 从小开始，逐步扩大
4. **Defined Timeline** — 典型 2-8 周
5. **Measurable Goals** — 具体目标

### 运营公司

1. **Review Plans Carefully** — 为一切奠定基础
2. **Monitor Early** — 密切关注前几个任务
3. **Build Knowledge Base** — 保存重要文档
4. **Check Costs Weekly** — 避免意外
5. **Communicate Regularly** — 使用 Chat 标签获取更新

### 批准工作

1. **Timely Reviews** — 不要让任务在人工审查中停留
2. **Clear Feedback** — 具体的更改请求
3. **Document Decisions** — 为批准添加注释
4. **Trust but Verify** — 抽查自主工作

## 常见工作流

### 启动活动

1. 创建 "March Campaign" 公司
2. 分配 Atlas 作为 CEO
3. 设定目标："Generate 100 leads"
4. Atlas 提交包含团队结构的计划
5. 批准计划
6. Atlas 聘请 Echo（Strategy）、Nova（Content）、Pixel（Design）
7. 任务出现在板上
8. 在 Knowledge 标签中审查交付物
9. 批准最终内容
10. 完成后归档公司

### 运行持续支持

1. 创建 "Customer Support" 公司
2. 设置永久状态（无结束日期）
3. Atlas 聘请支持 Agent
4. 连接到支持渠道
5. Agent 将工单作为任务处理
6. 从解决方案中建立知识库
7. 审查月度分析

## 故障排除

**Company Stuck in Planning：**

- CEO 可能需要更多信息
- 检查 Chat 标签中的问题
- 提供澄清
- 或编辑简报并重新提交

**Tasks Not Moving：**

- 检查 Agent 是否在线
- 验证任务未被阻塞
- 查看 Activity 标签中的错误
- 可能需要生成更多 Agent

**Costs Too High：**

- 查看 Costs 标签了解分解
- 考虑更便宜的 LLM 模型
- 减少 Agent 数量
- 设置更严格的预算限制

**Approvals Backing Up：**

- 定期审查 Approvals 标签
- 留出时间进行审查
- 考虑切换到自主模式
- 或减少招聘的 Agent
