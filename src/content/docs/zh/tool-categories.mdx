# 工具类别

CapiBot Agent 可以访问 25+ 个按类别组织的工具。了解可用工具有助于您有效委派工作。

## Agent 如何使用工具

Agent 根据需要什么自动选择工具：

```
You: Research top 5 competitors

Scout (Research Analyst):
  1. Analyzes request
  2. Decides to use web_search tool
  3. Searches for "top SaaS analytics competitors"
  4. Reads results
  5. Writes findings to file
  6. Reports back to you
```

**您不需要告诉 Agent 使用哪些工具** — 他们了解自己的能力并为工作选择正确的工具。

## 文件操作

读取、写入和管理文件。

### 可用工具

**`read_file`**

- 读取任何文件的内容
- 为大文件指定行偏移和限制
- 支持所有文本格式

**`write_file`**

- 创建新文件
- 覆盖现有文件
- 自动创建父目录

**`edit_file`**

- 外科手术式行级编辑
- 在文件中查找和替换
- 插入或删除特定行

**`list_dir`**

- 列出目录内容
- 查看文件和子目录
- 按模式筛选

### Agent 何时使用这些

| 任务 | 使用的工具 |
|------|-----------|
| 阅读您的说明 | read_file |
| 保存完成的工作 | write_file |
| 修复代码中的错误 | edit_file |
| 探索项目结构 | list_dir |
| 更新文档 | edit_file |

### 示例

```
You: Write a Python script to analyze CSV data

Nova: I'll create that script for you.

[Uses write_file to create analyze_csv.py]

Done! I've created analyze_csv.py with functions
for loading data, calculating statistics, and
generating charts.
```

### 可用工具

**`web_search`**

**`web_fetch`**

- 下载和解析任何网页
- 提取主要内容
- 处理各种格式

### Agent 何时使用这些

| 任务 | 使用的工具 |
|------|-----------|
| 研究竞争对手 | web_search |
| 阅读文档 | web_fetch |
| 查看当前新闻 | web_search |
| 收集参考资料 | web_fetch |
| 查找示例 | web_search |

You: What are the latest AI trends in 2024?

Scout: Let me search for current AI trends.

1. Multimodal AI models
2. AI agents and automation
3. Edge AI deployment
4. AI regulation and compliance

Would you like me to dive deeper into any of these?

## Shell 与系统

执行命令和管理进程。

### 可用工具

**`exec`**

- 运行 shell 命令
- 危险模式自动阻止
- 可配置超时（默认 30s）
- 每次调用输出限制约 10KB

**`spawn`**

- 启动后台进程
- 长时间运行任务
- 进程管理

### 安全功能

**阻止的命令：**

```bash
# These are automatically blocked:
rm -rf /
mkfs.*
dd if=/dev/zero
fork bombs
network scans
privilege escalation
```

**默认安全：**

- 工作区限制
- 超时限制
- 输出限制
- 命令日志

### Agent 何时使用这些

| 任务 | 使用的工具 |
|------|-----------|
| 运行 Python 脚本 | exec: python script.py |
| 安装依赖 | exec: npm install |
| Git 操作 | exec: git commit -m "..." |
| 构建项目 | exec: npm run build |
| 数据库迁移 | exec: migrate up |

### 示例

```
You: Run the test suite

Nova: Running tests now.

[Uses exec: npm test]

✅ 45 tests passed
⚠️  2 tests skipped
❌ 0 tests failed

All tests passing! The build is ready for deployment.
```

## 工具安全

### Agent 不能做什么

**自动阻止：**

- 破坏性系统命令
- 工作区外访问
- 超过超时的命令
- 已知的危险 shell 模式

### 您可以控制什么

**每个 Agent 设置：**

- 启用/禁用特定工具
- 限制文件系统访问
- 设置命令超时
- 限制输出大小

**全局设置：**

- 工作区限制
- 危险模式阻止
- 命令日志
- 审计追踪

## 最佳实践

### 对于您

1. **Trust Agents** — 他们知道使用哪些工具
2. **Review Deliverables** — 检查他们创建的文件
3. **Ask for Explanations** — "How did you do that?"
4. **Set Boundaries** — 需要时限制

### 对于 Agent

1. **Use Right Tool** — 工具与任务匹配
2. **Handle Errors** — 优雅失败
3. **Report Actions** — 告诉您他们做了什么
4. **Respect Limits** — 在约束内工作

## 故障排除

**Agent Says "I Can't Do That"：**

- 工具可能被禁用
- 超出其能力
- 安全限制
- 检查 Agent 的允许工具

**Command Failed：**

- 检查错误消息
- 可能需要不同方法
- 可能是权限问题
- 尝试拆分为更小的步骤

**File Not Found：**

- 检查路径是否正确
- 可能需要创建父目录
- 验证文件存在
- 检查权限
